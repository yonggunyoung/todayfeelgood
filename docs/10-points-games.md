# 10. 포인트·미니게임·토스 입점 — 설계와 운영

> 목표: 절약 "행동"을 보상해 매일 열어보게 만들고(리텐션), 짬시간에 광고 수익을 만든다.
> 같은 코드가 **개별 운영(웹/PWA)** 과 **앱인토스(토스 미니앱)** 양쪽에서 돈다.

## 1. 포인트 경제 (js/points.js)

원칙: **무상 적립만** (현금 충전 없음 → 선불전자지급수단 규제 비대상), 현금 환급 없음,
일일 상한으로 어뷰징 차단, 교환은 내부 보상(원가≈0)부터.

| 적립처 | P | 일일 상한 | 비고 |
|---|---|---|---|
| 출석 | 5 | 1회 | 앱 첫 실행 시 자동 |
| 요리 완료 | 10 | 3회 | 재고 차감 확정 시 |
| 임박 재료 구출 | 20 | 3회 | D-3 이하 재료를 요리에 사용 |
| 잔반 해결 | 15 | 3회 | 잔반 "먹었어요" |
| 광고 시청 보너스 | 3 | 3회 | AI 충전 광고 완주에 얹어줌 |
| 미니게임 | 점수 기반 1~12 | 5판 | 판수 초과 시 기록 도전만 가능 |

→ 헤비 유저 일일 최대 약 150P. 하루 광고 3완주 수익(약 30~90원) 대비 내부 보상 원가는 0에 수렴.

### 교환소 (SHOP)

| 상품 | P | 원가 | 구현 |
|---|---|---|---|
| AI 1회권 | 100 | ~7원 | 서버 `/redeem` (일 2회 상한, REDEEM_DAILY_CAP) |
| 광고 없는 하루 | 300 | 0 | 클라이언트 `adFreeUntil` — 배너 3슬롯 미노출 |
| 프리미엄 맛보기 1일 | 1,000 | 0 | `planTrialUntil` — 광고 없음 체험 → 구독 전환 유도 |
| 기프티콘 (잠금) | 5,000 | 실비 | P1: 광고 실수익 확인 후 B2B 발행사 연동 |
| 토스포인트 전환 (잠금) | — | 실비 | P2: 앱인토스 프로모션으로 제공 |

예산 수칙: 외부(실비) 교환을 열 때는 **월 광고 수익의 15~20%만 포인트 예산**으로 — 수익 연동 상한이라 역마진 불가.

## 2. 미니게임 (js/games.js) — 짧고, 중독성, 승부욕

| | 프레시 캐치 (원터치) | 외쳐! 재료 (음성인식) |
|---|---|---|
| 룰 | 움직이는 마커를 초록 "신선 존"에서 탭 — 가운데 40%는 PERFECT | 재료 이모지 보고 이름을 외치기 — 8문제 스피드런 |
| 중독 장치 | 콤보 ×5까지, 라운드마다 빨라지고 존이 좁아짐+위치 랜덤, 생명 3개 | 60초 예산, 4초 막히면 초성 힌트, 패스 +8초 페널티 |
| 승부욕 | 역대 최고 + **주간 기록(월요일 리셋)** | 동일 |
| 판당 시간 | 20~40초 | 30~60초 |
| 지원 | 전 기기 | 크롬/안드로이드 (아이폰 사파리는 허브에 잠금 표시) |

수익 연결 (전부 `playAd` 코어 경유 — 광고망 교체 1곳):
- 판 종료 → 포인트 적립 → **[📺 광고 끝까지 보고 2배 받기]** (완주해야 지급)
- 입구: 홈 포인트 카드 · **요리 타이머 칩**("⏲ 7:32 🎮 기다리는 동안 한 판?" — 짬시간 천연 슬롯)
- 일 5판 보상 제한 → 광고 시청 동기는 유지하되 무한 파밍 차단

## 3. 토스 입점 어댑터 (js/toss.js)

같은 빌드가 토스 안에서 자동 전환된다 — SDK 전역이 감지되면:

| 기능 | 개별 운영 (지금) | 앱인토스 |
|---|---|---|
| 보상형 광고 | 하우스 15초 (AdFit 교체 자리) | `showFullScreenAd` → `userEarnedReward` 수신 시 보상 |
| 포인트 | 앱 내 포인트 | 앱 내 포인트 + **토스포인트 프로모션 미러링** (`tossGivePoints`) |
| 결제 | 토스페이먼츠 (G2) | 토스 인앱결제 SDK |

### 입점 체크리스트 (운영자)

1. 앱인토스 콘솔 가입 (만 19세+, 본인 명의 토스앱 — 사업자 없이 출시 가능)
2. 미니앱 등록: 이름 냉비서 / 카테고리 라이프 / 개인정보처리방침 URL(blog/privacy.html) / 고객센터 채널
3. 콘솔에서 **보상형 광고 단위** 생성 → `js/config.js` `TOSS.rewardAdId`
4. 콘솔에서 **프로모션(토스포인트)** 생성 + **비즈월렛 예산 선충전** → `TOSS.promotionId`
   - 지급 실비만 차감, 플랫폼 수수료 없음 · 예산 80% 소진 시 메일 알림 · 소진 시 지급 자동 중단
5. WebView SDK에 앱 URL 연결 → 검수 제출 (비게임 1~3개월)
6. SDK 전역 객체/함수명은 입점 시점 문서로 확정 → `js/toss.js`의 후보 명칭 수정 (30분 작업)

## 4. 법·운영 체크 (요약)

- 적립형 포인트(무상)·내부 보상 교환: 규제 부담 없음. **현금 충전/환급은 도입하지 않는다.**
- 기프티콘 등 경품: 1인당 5만원 초과 시 제세공과금(기타소득 22%) 이슈 — 소액 상품만.
- 어뷰징: 일일 상한(클라) + `/redeem`·`/reward` 서버 상한(일 2/3회) 이중. 외부 교환 도입 시 본인인증 추가.
- 게임은 실력 기반(사행성 아님). 룰렛 등 추첨형을 추가할 땐 "꽝 없음 소액"으로 설계.
