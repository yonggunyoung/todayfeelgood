# 08. 커뮤니티 전략 — 별도 사이트 × 앱 연동

> 질문: "별도 커뮤니티 웹을 만들어 레시피 자랑·추천 등으로 운영하고, 앱과 상호작용(검색→레시피 적용)하며,
> 레시피+생활정보로 운영해 쇼핑·검색노출·디스커버 노출까지 노린다 — 이건 괜찮은가?"

## 1. 평가: 방향은 맞다 — 단, "앱의 부속"이 아니라 "유입 엔진"으로 설계해야 성립

| 관점 | 판정 | 근거 |
|---|---|---|
| 전략 자체 | ✅ 좋음 | 앱(PWA)은 검색엔진에 거의 안 잡힌다. 콘텐츠 사이트가 검색·디스커버 유입을 만들고 앱으로 흘려보내는 구조는 검증된 패턴 (만개의레시피·우리의식탁이 이 구조) |
| 수익 시너지 | ✅ 좋음 | 커뮤니티 글마다 재료 → 쿠팡 파트너스 링크 = 콘텐츠가 곧 수익 지면. 애드센스 승인도 앱보다 사이트가 쉬움 |
| 앱 연동 | ✅ 이미 절반 완성 | 공유 링크(`?share=NB1…`) 체계가 있어서, 커뮤니티 글의 **[🧊 내 냉장고에 담기]** 버튼은 링크 한 줄이면 된다 |
| 시점 | ⚠️ 주의 | 커뮤니티는 "만들면 모이는" 게 아니라 "콘텐츠 시드를 계속 붓는" 운영 노동. 앱 리텐션 검증(G1) 전에 벌이면 둘 다 어중간해진다 |
| 빈 광장 리스크 | ⚠️ 주의 | 글 0개인 게시판은 역효과. 초기엔 "커뮤니티"가 아니라 **운영자 블로그/매거진**으로 시작해야 한다 |

**결론: 채택하되 3단계로.** 처음부터 게시판을 열지 말고, ① 운영자 콘텐츠(매거진) → ② 제보·참여(반쯤 커뮤니티) → ③ 회원 게시판(진짜 커뮤니티) 순서.

## 2. 단계 설계

### C1 — 매거진 (G1과 병행 가능, 운영자 1인)
- **형태**: SEO 되는 블로그형 사이트. 글 유형: 레시피(앱 담기 버튼 포함), 재료 보관법, 냉파 절약 팁, 자취 생활정보
- **글 = 구조화 데이터**: 각 레시피 글에 Recipe 스키마(JSON-LD) 심어 구글 레시피 리치결과·디스커버 노출 노림. 대표 이미지 1200px 이상(디스커버 조건)
- **앱 연동**: 글마다 `[🧊 내 냉장고에 담기]` = `https://앱주소/?share=NB1…` — 탭 한 번에 앱에 레시피 등록 (이미 작동하는 기능)
- **수익**: 본문 재료 리스트에 쿠팡 파트너스 링크, 사이트 애드센스
- **호스팅**: GitHub Pages 정적으로도 가능(글이 HTML로 빌드되면 SEO OK) — 단, 규모 커지면 Next.js+Vercel 권장
- 주 2~3편 × 3개월 = 글 30편이 검색 유입의 임계점

### C2 — 참여형 (앱 MAU 1천~)
- 독자 레시피 제보(구글폼/카카오채널) → 운영자가 다듬어 게재("이달의 독자 레시피")
- 앱에서 "내 레시피 자랑하기" 버튼 → 공유 링크 + 완성 사진을 제보폼으로 전송
- 좋아요/조회수 노출 시작 (Firestore 카운터 — 로그인 없이 가능)

### C3 — 진짜 커뮤니티 (MAU 5천~, G3)
- 회원 게시판: 레시피 자랑 / 오늘 뭐 해먹지 Q&A / 냉파 챌린지 / 절약 인증
- 글의 레시피는 자동으로 공유 코드 생성 → 모든 글이 "앱에 담기" 가능
- 앱 안 "둘러보기" 탭 신설 ← 커뮤니티 인기글 API로 표시 (검색·적용·상호작용)
- 필요 인프라: 로그인(카카오/구글), 신고·운영 도구, 운영지침 — **운영 코스트가 본격 발생하는 지점**

## 3. 기술 스케치

```
커뮤니티(콘텐츠) 사이트            앱 (PWA)
  글 + Recipe JSON-LD   ←링크→   ?share=NB1… 가져오기 (구현됨)
  쿠팡 파트너스/애드센스           둘러보기 탭(C3): 인기글 fetch
  Firestore (좋아요·댓글 C2~)  ←  같은 Firebase 프로젝트 공유
검색/디스커버/SNS → 사이트 → 앱 설치 → 리텐션 → 구독/파트너스
```

도메인을 산다면: `예시.kr`(사이트) + `app.예시.kr`(앱) — 같은 브랜드로 묶고 SEO 권위도 한 도메인에 적립.
