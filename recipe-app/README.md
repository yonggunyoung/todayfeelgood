# 레시피북 (MVP)

> 어디서 본 레시피든, 붙여넣기 하나로 내 레시피북에.

유튜브에서 본 요리 레시피를 매번 다시 검색해야 하는 불편을 해결하는 웹앱입니다.
링크/텍스트를 붙여넣으면 AI가 **재료 목록 + 조리 단계(영상 타임스탬프 포함)** 구조로
정리해 저장하고, 요리할 때 검색·재료 체크·단계별 영상 점프로 활용합니다.

## 실행

```bash
npm install
npm run dev      # 개발 서버
npm run build    # 프로덕션 빌드
```

## 현재 지원 소스 (Phase 1)

| 소스 | 동작 |
|------|------|
| 유튜브 링크 | 메타데이터(제목·썸네일) 자동 + 자막/설명 붙여넣기 → AI 구조화. 상세 화면에서 공식 임베드 재생, 단계별 타임스탬프 점프 |
| 웹/블로그 링크 | 본문 자동 추출(r.jina.ai) 시도 → AI 구조화. 실패 시 본문 붙여넣기 |
| 텍스트 | 붙여넣은 내용 그대로 AI 구조화 |

## AI 변환

- 설정(⚙)에서 Claude API 키 입력 시 **실제 AI 변환** (structured outputs로 스키마 보장)
- 키가 없으면 **데모 변환**으로 화면 흐름 확인 가능
- 키는 브라우저 localStorage에만 저장 (MVP 전용 — 실서비스 전환 시 서버 프록시 경유로 변경 필요)

## 아키텍처

```
입력(링크/텍스트)
  → 소스 판별 라우터 (src/lib/extract.js)
  → 소스별 추출기 (유튜브 메타 / 웹 본문 / 텍스트)
  → AI 정규화 (src/lib/ai.js) → 표준 레시피 스키마 (src/lib/recipes.js)
  → 레시피북 (검색·태그·localStorage 저장)
```

모든 소스가 하나의 표준 스키마로 수렴하므로, 이후 단계(냉장고 재료 매칭,
장보기 연동, 커뮤니티 공유)는 전부 이 스키마 위에 쌓입니다.

## 로드맵

- **Phase 1 (현재)**: 유튜브/웹/텍스트 → AI 레시피 변환 + 저장/검색
- **Phase 1.5**: 이미지/스크린샷 비전 AI 추출(인스타·요리책 커버), 계정 동기화(Firebase)
- **Phase 2**: 냉장고/재료 관리 + "지금 있는 재료로 만들 수 있는 레시피" 추천, 장보기 제휴 링크
- **Phase 3**: 레시피북 공유·커뮤니티
