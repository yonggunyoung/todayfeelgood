# 냉장고 관리 + 레시피 추천 앱 — 제품 설계 (0→100)

> 가칭: **냉비서** (후보: 냉파고 / 오늘냉장고 / Fridgy — "냉장고를 부탁해"는 JTBC 상표라 사용 금지)
> 작성일: 2026-06-10 · 문서 구성: 00 개요 → 01 UX 플로우 → 02 AI 설계 → 03 수익화 → 04 기술 아키텍처 → 05 로드맵

## 1. 한 줄 정의

**내 냉장고를 거의 자동으로 관리해주고, 내 목표(운동·산모·자린고비)에 맞춰 "지금 뭐 해먹을지"를 답해주는 앱.**

## 2. 해결하는 문제

| 문제 | 현재 사용자 행동 | 우리의 답 |
|---|---|---|
| 냉장고에 뭐가 있는지 모름 | 열어보고 까먹고 또 사옴 | 영수증/사진 1장으로 재고 자동 입력 |
| 재료를 썩혀서 버림 | 유통기한 지나 폐기 (돈 낭비) | 소비기한 알림 + 임박 재료 우선 레시피 |
| 오늘 뭐 해먹을지 모름 | 배달 시킴 | 보유 재료 + 목표 모드 기반 추천 |
| 요리 후 재고가 어긋남 | 관리 포기 (기존 앱들의 사망 원인) | 요리 완료 원탭 자동 차감 |
| 남은 음식을 까먹고 버림 | 냉장고 안에서 화석화 | 잔반 타이머 + "먼저 먹기" 추천 |
| 떨어진 재료 재구매 귀찮음 | 마트 가서 또 까먹음 | 장보기 리스트 자동 생성 + 원탭 쿠팡 |

## 3. 타겟과 모드 (핵심 차별화)

같은 냉장고를 두고 **목표 모드에 따라 다른 추천**을 주는 것이 경쟁 앱(냉파, 오늘의냉장고, 레시피오 등)과의 가장 큰 차별점이다. 기존 앱들의 추천은 전부 "있는 재료로 만들 수 있는 요리" 한 종류뿐이다.

| 모드 | 타겟 | 추천 로직 | 핵심 지표 표시 |
|---|---|---|---|
| **운동 모드** | 헬스인, 다이어터 | 단백질 우선 정렬, 매크로 목표 반영 | 단백질 g / 칼로리 |
| **자린고비 모드** | 자취생, 알뜰족 | 소비기한 임박 재료 우선 소진, 추가 구매 0원 우선 | 이번 달 아낀 돈 / 버린 돈 |
| **산모 모드** | 임산부, 수유부 | 영양 보충 + **금기·주의 식재료 하드 필터** | 주의 재료 경고 |
| (확장) 이유식·당뇨식·다이어트 | 주부, 환자 가족 | 모드 = 프로필 설정값이라 추가 비용 거의 0 | — |

모드는 추천 엔진 하나에 프로필만 갈아끼우는 구조라서, 모드 추가 = 콘텐츠 큐레이션 작업이지 개발 작업이 아니다. (상세: `02-ai-design.md`)

## 4. 핵심 루프 (제품의 심장)

```
[입고] 장보기 → 영수증/사진 1장 → AI가 재고 등록
   ↓
[보관] 소비기한 자동 추적 → "계란 3일 남음" 알림
   ↓
[추천] 모드 + 보유 재료 → "오늘 이거 해먹어요" (임박 재료 우선)
   ↓
[조리] 레시피 보며 요리 → "요리 완료" 원탭 → 재고 자동 차감
   ↓
[잔반] 남은 음식 등록 → 타이머 → 다음 끼니 1순위 추천
   ↓
[재구매] 떨어진 재료 → 장보기 리스트 → 쿠팡 파트너스 원탭 (수익 발생)
   └──→ 다시 [입고]
```

루프가 한 바퀴 돌 때마다: 사용자는 끼니가 해결되고, 데이터(소비 패턴)가 쌓이고, 우리는 파트너스 수익 기회를 얻는다.

## 5. 경쟁 대비 포지셔닝

| | 기존 냉장고 앱 (냉파 등) | 영상 레시피 앱 (해먹으리 등) | **우리** |
|---|---|---|---|
| 재고 입력 | 수동 (포기율 높음) | 없음 | **AI 사진/영수증 자동** |
| 추천 | 단일 ("만들 수 있는 요리") | 없음 (저장만) | **모드별 목표 추천** |
| 차감 | 수동 | 없음 | **요리 완료 원탭 자동** |
| 잔반 관리 | 없음 | 없음 | **타이머 + 우선 추천** |
| 수익모델 | 광고 | AI 추출 구독 | **구독 + 파트너스 이중** |

## 6. 설계 3원칙 (모든 문서에 적용)

1. **코어 루프는 AI 없이도 완결된다.** AI는 마찰을 줄이는 가속기다. AI 한도가 떨어져도 앱은 멀쩡히 돌아가야 한다. (변동비 통제 + 무료 사용자 경험 보장)
2. **정밀함보다 꾸준함.** 그램 단위 재고 추적은 UX의 무덤이다. 목적은 회계장부가 아니라 "장보기 신호"와 "폐기 방지 신호"다. 퍼지 수량(개수/레벨)으로 충분하다.
3. **사용자 행동 1회 = 탭 1~2회.** 입고는 사진 1장, 차감은 버튼 1탭, 잔반은 분기 1탭. 그 이상 요구하면 3주 안에 삭제된다.
