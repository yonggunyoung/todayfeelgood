# 09. 상용화 런북 — API·광고·결제·앱인토스 실행 절차

> 코드는 전부 준비돼 있고, 이 문서는 "계정을 만들고 스위치를 켜는" 운영자 행동 순서다.
> 순서 중요: **① 서버 AI → ② 광고 → ③ 결제 → ④ 앱인토스** (각 단계가 다음 단계의 전제)

## ① API(서버 AI) 적용 — 소요 ~30분, 선행조건 없음

사용자가 키 없이 AI를 쓰게 하는 스위치. 코드: `functions/` (완성), 가이드: docs/07.

1. console.anthropic.com → 가입 → 결제수단 등록(또는 크레딧 $5~) → **API Key 발급**
2. Firebase 콘솔 → (기기연동 때 만든) 프로젝트 → **Blaze(종량제) 업그레이드** (외부 호출 필수 — 소규모는 실과금 거의 0)
3. 내 PC에서:
   ```bash
   npm i -g firebase-tools && firebase login
   git clone https://github.com/yonggunyoung/todayfeelgood && cd todayfeelgood
   firebase use <프로젝트ID>
   firebase functions:secrets:set ANTHROPIC_API_KEY     # 운영자 키 붙여넣기
   cd functions && npm install && cd ..
   YT_API_KEY=AIza… firebase deploy --only functions    # 유튜브 인앱검색 키(선택)까지 한 번에
   ```
4. 출력된 함수 URL → 앱 설정(관리자) → **☁️ 서버(유료화)** 선택 + URL 저장
5. 가드레일: Google Cloud 예산 알림(월 2만 원), Anthropic 콘솔 spend limit 설정
6. 검증: 새 기기에서 영수증 스캔 → 작동 + Firestore `ai_usage`에 카운트 1 확인. 11회째에 충전 시트 뜨면 정상

## ② 광고 적용 — 보상형 슬롯에 실광고 꽂기

현재: 한도 소진 → 충전 시트 → 15초 하우스 광고 → 서버가 +1 충전(일 3회 상한). **이 퍼널은 이미 작동** — 광고 네트워크만 갈아끼우면 매출화된다.

| 단계 | 네트워크 | 신청 | 우리 쪽 작업 |
|---|---|---|---|
| 웹(지금) | 카카오 AdFit | adfit.kakao.com — 사이트 등록, 심사 수일 | `UI.watchAd()`의 하우스 카드 자리에 AdFit 배너 스크립트 삽입 (15초 노출 유지) |
| 웹(병행) | Google AdSense | 콘텐츠 필요 → **매거진(/blog)이 심사용 자산** | 매거진에 자동광고 + 앱 충전 슬롯 |
| 앱인토스 | 토스 인앱 광고 SDK | ④ 입점 후 SDK 연동 | 같은 슬롯에 토스 보상형 연결 |
| 스토어 앱 | AdMob 리워드 | Capacitor 래핑 후 | 진짜 보상형 영상 (eCPM 최고) |

운영 수칙: 광고는 **충전 슬롯에만** (요리 화면 배너 금지 — docs/03 원칙), 일 3회 상한 유지, 광고 단가 < AI 원가가 되면 한도/충전비율 조정.

## ③ 결제 시스템 + 운영 — 구독 매출 켜기

전제: 사업자등록 (홈택스, 간이과세 1일) → 통신판매업 신고 (정부24, ~수일).

**구조 (웹 결제 — 수수료 최소 경로)**
```
앱 [프리미엄] 버튼 → 결제 페이지(pay.html, 토스페이먼츠 위젯)
→ 성공 webhook → Functions가 Firestore users/{uid}.plan='premium' 기록
→ 앱·서버가 plan 확인: consumeQuota에서 프리미엄이면 한도 통과 (코드 한 줄 분기 — 자리 마련돼 있음)
```
1. 토스페이먼츠(tosspayments.com) 또는 포트원 가입 — 사업자번호 필요, 정기(빌링) 결제 신청
2. 결제 페이지 1장 + 성공 콜백 Functions 엔드포인트(`/subscribe`) 추가 ← 이 부분은 G2 진입 시 내가 구현
3. 가격: 월 3,900 / 연 29,000 (docs/03 A/B 계획)
4. 운영 루틴(주 1회): Firestore `ai_usage` 사용량·`premiumWish` 수 확인, 광고 충전률, 해지 사유 수집
5. 환불·해지: 약관 페이지 필수 (전자상거래법 — 구독 7일 내 미사용 환불 등 표준 문구)

**대안 경로**: 앱인토스 안에서는 토스 인앱결제 SDK로 동일 구독 판매 (정산은 토스 수수료 차감 — 입점 후 약관 확인)

## ④ 앱인토스(토스 미니앱) 생성 — 상세

조사 기준(2026-06, 공식 개발자센터):

1. **가입**: [앱인토스 콘솔](https://developers-apps-in-toss.toss.im/prepare/console-workspace.html) 회원가입 — 만 19세 이상 + 본인 명의 토스앱 필요. **사업자 없이도 출시 가능** (인앱결제·정산 쓰려면 사업자 정보 등록)
2. **워크스페이스 생성** → 미니앱 등록: 로고, 앱 이름(냉비서), 카테고리(라이프), 고객센터 정보
3. **개발 방식**: WebView 또는 React Native 기반 SDK 제공 — 우리는 순수 웹앱이라 **WebView SDK에 현재 앱 URL을 올리는 방식**이 기본. SDK로 연동하는 것: 인앱 결제, 인앱 광고, 푸시 알림, (게임용 리더보드)
   - 우리 쪽 수정 예상: 토스 SDK 초기화 스크립트 + 토스 결제/광고를 기존 슬롯(충전 시트·프리미엄 버튼)에 연결 + 토스 디자인 가이드 점검 — 예상 공수 1~2주
   - 선행 필수: **① 서버 AI** (토스 사용자에게 키 입력 불가) — 그래서 이 문서 순서가 ①→④
4. **검수 요청**: 콘솔에서 제출 — 비게임 서비스 약 **1~3개월** 소요(게임은 2~4주). 검수 대비: 개인정보처리방침 URL, 고객센터 채널(카카오톡 채널 추천), AI 결과 신고 버튼
5. **수익 구조**: 토스가 인앱 결제·광고 수익 일부를 수수료로 가져가는 대신 토스 유저 풀에 노출 — 정확한 요율은 콘솔 약관에서 확인 후 docs/03 단가에 반영
6. 출시 후: 토스 검색/추천 노출 → 매거진·뚝딱과 함께 3개 유입구 완성

## 전체 타임라인 제안

```
이번 주      ① 서버 AI 켜기 + 쿠팡 파트너스·AdFit 신청 + 도메인
~2주        지인 베타 지표 수집 · 매거진 글 추가 (애드센스 심사 대비)
지표 확인 후  ③ 사업자·PG → 결제 구현(내 작업) → 구독 오픈
결제 가동 후  ④ 앱인토스 등록 → SDK 연동(내 작업) → 검수 제출 (비게임 1~3개월 감안해 일찍)
```

## 프리미엄 권한 동작 (구현됨)

- **무료**: AI 월 10회 + 광고 시청 충전(하루 3회) — 광고는 이 충전 슬롯에서만 노출
- **프리미엄**: 한도 검사 통과 = **광고 볼 일 자체가 없음** + AI 전 기능(스캔·유튜브 정리·검색) 제한 없이 사용.
  남용 방지 상한만 월 200회(환경변수 PREMIUM_QUOTA)로 존재하며 도달 시 안내 문구가 다름
- **부여 방법**: 결제 연동(G2) 전에는 Firestore → `users/{uid}` 문서에 `plan: "premium"` 수동 기록으로
  즉시 적용 (만료일을 두려면 `premiumUntil: <Timestamp>` 추가 — 없으면 영구). 결제 연동 시 PG 웹훅이
  같은 필드를 쓰면 끝이라 구조 변경 없음
