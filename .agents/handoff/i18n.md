# i18n / 국제화 — FE-Agent 인수인계

한 줄 결론: 홈·폰트 랜딩을 한국어(기존 경로)/영어(`/en`) 이중 운용 + 다국어 SEO(hreflang·og:locale·`<html lang>`·sitemap) 적용 완료. `pnpm -r build` / `pnpm -r lint` 모두 통과.

## 라우팅
- home: KO=`/`, EN=`/en`.
- font(basePath `/font`): KO=`/font`, EN=`/font/en`.
- 구현: 각 앱 `app/` 아래 **route group**으로 분리 — `(ko)`(루트 레이아웃 `lang=ko` + 기존 페이지/studio/s), `(en)`(루트 레이아웃 `lang=en` + `/en` 페이지). 여러 root layout으로 `<html lang>`을 로케일별로 정확히 렌더.
- 기존 KO URL(`/`, `/font`, `/font/studio`, `/font/s`)은 변경 없음. studio/s는 이번 번역 범위 밖이라 KO 기본 문자열 유지.

## 사전(dictionary)
- `home/lib/i18n.ts` + `home/lib/dictionaries/{ko,en}.ts`
- `apps/font/frontend/lib/i18n.ts` + `lib/dictionaries/{ko,en}.ts`
- `ko`가 타입 원본(`Dictionary = typeof ko`), `en: typeof ko`로 키 누락 시 빌드 에러. 문자열 키로 관리, 언어 추가는 `SUPPORTED` + dictionaries 확장.
- 렌더는 `components/HomeView.tsx`(home) · `components/LandingView.tsx`(font)가 locale 받아 사전으로 그림. KO/EN 페이지가 공유.

## EN 카피(마케팅 영어, 기계번역 아님)
- 피치: "Turn your handwriting into a real font — free, no AI." 한글 강점은 "Korean & Latin supported".
- 마스코트(Neoguri) 유지, "~너굴" 말장난은 EN에서 생략(담백).

## SEO (`packages/seo/src/index.ts`)
- `buildMeta`를 **locale 파라미터화**(`locale?: "ko"|"en"`, 기본 ko → 하위호환). `og:locale`(ko_KR/en_US), `alternates.languages`(hreflang) 지원.
- `htmlLang(locale)` export → 레이아웃 `<html lang>`(ko-KR/en-US)·JSON-LD `inLanguage`.
- `keywords` 타입을 `readonly string[]`로 완화(dictionaries의 배열 수용). 내부에서 mutable 복사.
- 각 페이지 `alternates.languages`에 ko/en/x-default. sitemap(home·font)도 ko/en + xhtml:link 대체 + studio. robots 그대로.
- 페이지별 EN title/description/keywords: "handwriting font generator", "draw your own font", "free handwriting to font" 등.

## 언어 전환 UI
- `components/LanguageToggle.tsx`(+ .module.css) 각 앱. KO/EN `<a>` 토글, `aria-current`/`hrefLang`/`lang` 속성, nav aria-label. 현재 페이지의 ko/en 경로로 전환.
- home: 상단바(`.topbar`). font: SiteHeader `right` 슬롯(스튜디오 CTA 옆).

## basePath 규칙 유지
- 앱 내부 이동은 `next/Link`(자동 `/font` prefix): studio CTA = `/font/studio` 확인.
- 홈 허브로 가는 링크는 일반 `<a>` 절대경로(basePath 안 붙음): font EN 헤더 brand → `/en` 확인.

## 검증 결과
- 빌드 HTML 확인: `/`·`/en`·`/font`·`/font/en` 모두 `<html lang>`/`og:locale`/canonical/hreflang(ko-KR·en-US·x-default) 정상. EN 페이지 영어 카피·토글 렌더 확인. sitemap 양 언어 + alternates 출력.
- `pnpm -r build`, `pnpm -r lint` 통과(전 워크스페이스).

## 후속(범위 밖)
- 스튜디오/공유(s) 내부 전체 문자열 EN 번역은 미적용(랜딩/홈 우선). 필요 시 같은 사전 패턴으로 확장 가능.
