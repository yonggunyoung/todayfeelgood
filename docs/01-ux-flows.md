# 01. UX 플로우 상세

핵심 질문에 대한 답: **"레시피 선택 → 재료 꺼내 사진 → 자동 차감"을 더 간편하게 개선한 설계.**

## A. 입고 플로우 (재고 등록)

입력 수단 4종. 마찰이 낮은 순서로 사용자에게 노출한다.

| 수단 | 방법 | AI | 정확도 | 커버 범위 |
|---|---|---|---|---|
| **영수증 스캔** (대표) | 영수증 사진 1장 → 품목/수량 파싱 → 확인 화면 → 일괄 등록 | O | 높음 | 마트 장보기 전체 |
| **장본 것 사진** | 식탁에 펼쳐놓고 1장 → 품목 인식 | O | 중 | 영수증 없을 때 |
| **주문내역 캡처** | 쿠팡/컬리 주문내역 스크린샷 → 파싱 | O | 높음 | 온라인 장보기 |
| **퀵추가** (무료 기본) | 검색 + 최근/즐겨찾기 품목 → 2탭 등록 | X | 정확 | 낱개 추가, AI 한도 소진 시 |

등록 시 자동 처리 (전부 비AI):
- **재료 정규화**: "서울우유1L" → 마스터 재료 "우유" 매핑 (사전 + 동의어 테이블)
- **소비기한 자동 제안**: 품목별 기본값 사전 (우유 +10일, 계란 +30일, 시금치 +5일…) — 사용자는 수정만
- **보관 위치**: 냉장/냉동/실온 기본값 자동

확인 화면 원칙: AI 파싱 결과는 **항상 수정 가능한 리스트로 보여주고 "전체 등록" 원탭**. 틀린 항목만 스와이프 삭제/수정.

## B. 조리·차감 플로우 ★ 사용자 제안의 개선안

### 원안의 문제
"재료 꺼내서 사진 촬영 → AI가 차감"은 (1) 요리 직전엔 손이 바쁘고 젖어 있어 마찰이 크고, (2) 매 요리마다 AI 비용이 들고, (3) 사진으로는 사용량(간장 30ml 등)을 알 수 없다.

### 개선: 3단 차감 체계 — "아는 정보는 묻지 않는다"

**1단 · 레시피 연동 차감 (기본 경로, AI 0원, 1탭)**

앱에서 레시피를 보고 요리했다면 무엇을 얼마나 썼는지 이미 안다. 사진이 필요 없다.

```
레시피 상세 → [요리 시작] → (조리 화면: 단계별 보기, 화면 꺼짐 방지)
→ [요리 완료] → 차감 미리보기 시트:
     인분 슬라이더 [1][2][3][4]  ← 레시피 기준 자동 환산
     ─ 계란 2개      (10 → 8)    [수정]
     ─ 양파 0.5개    (2 → 1.5)   [수정]
     ─ 간장          레벨 유지     [수정]
→ [확인] 1탭 → 차감 완료 → 잔반 분기로 이동
```

**2단 · 퍼지 수량 모델 (차감의 기반 규칙, AI 0원)**

| 수량 타입 | 대상 | 차감 방식 |
|---|---|---|
| 개수형 | 계란, 양파, 두부, 캔 | 레시피 분량만큼 숫자 차감 |
| 레벨형 (가득/반/조금/없음) | 간장, 기름, 쌀, 소스류 | 자동 차감 안 함. 임계 사용 횟수마다 "간장 아직 충분해요?" 1탭 질문 |
| 묶음형 | 대파 1단, 삼겹살 600g | 0.25 단위 분수 차감 |

정밀 그램 추적은 의도적으로 포기한다. 목적은 **장보기 신호**("간장 떨어져감")와 **폐기 방지 신호**("시금치 3일 남음")이지 회계가 아니다.

**3단 · 사진 차감 (예외 경로, AI 사용)** ← 원안 아이디어가 사는 곳

- **즉흥 요리**: 레시피 없이 요리한 경우 → "뭐 만드셨어요?" → 조리 후 사진 1장 or 텍스트 입력 → AI가 사용 재료 추정 → 차감 제안 리스트 → 확인
- **월 1회 정합성 체크 (프리미엄 기능)**: 냉장고 칸 사진 → AI가 현재 재고와 대조 → "등록엔 있는데 안 보여요: 두부" 보정 제안 → 일괄 반영

이렇게 하면 일상 90%(레시피 따라 요리)는 0원·1탭으로 돌고, 사진·AI는 예외 보정에만 쓰여 비용과 마찰이 동시에 최소화된다.

## C. 잔반(남은 음식) 플로우

요리 완료 직후 분기 1탭:

```
[요리 완료] → "다 드셨어요?"
   ├─ [다 먹었어요] → 끝 (끼니 기록 자동)
   └─ [남았어요] → 잔반 등록 시트 (요리명 자동 입력됨):
        보관: [냉장(기본)] [냉동]
        소비기한: 자동 3일 (냉동 30일) — 조정 가능
        사진: 선택사항
        → [저장] 1탭
```

잔반의 생애주기:
- **홈 화면 최상단 "먼저 먹기" 섹션 고정**: "어제 만든 카레 — 2일 남음 🔥"
- 추천 엔진에서 **잔반은 항상 1순위**: "오늘 점심: 남은 카레 + 계란프라이 (5분)"
- 임박 D-1 푸시: "내일까지! 김치찌개 남아있어요"
- 처리 버튼 2개: [먹었어요] → 끼니 해결 + 절약 집계 / [버렸어요] → 폐기 기록

폐기 기록은 숨은 자산이다 → 자린고비 모드의 "이번 달 버린 돈 ₩4,500 / 아낀 돈 ₩31,000" 리포트가 되고, 이 숫자가 리텐션과 구독 전환의 정서적 트리거가 된다.

식재료 잔반(반 쓴 양파, 반 남은 두부)은 별도 개념 없이 2단 퍼지 수량(0.5개)으로 자연 처리된다.

## D. 추천 플로우

홈 화면 = 답부터 보여준다:

```
①  먼저 먹기 (잔반 + 임박 재료)          ← 항상 최상단
②  오늘의 추천 3개 (모드 반영)
     [운동] 닭가슴살 덮밥 — 단백질 41g · 재료 다 있음
     [운동] 두부 스크램블 — 단백질 28g · 양파만 사면 됨
③  AI 셰프에게 물어보기 (프리미엄 진입점)
     "남은 두부랑 김치로 색다른 거 없을까?"
④  장보기 리스트 (부족 재료 자동 집계)
```

- ①② 는 비AI (정렬·필터, `02-ai-design.md` 참고) — 무료, 무제한, 즉시
- ③ 은 AI 생성형 — 한도제, 프리미엄 차별화 포인트

## E. 장보기·재구매 플로우 ("자동주문"의 현실 구현)

한국에는 제3자 소비자 주문 API가 없으므로 (쿠팡 Open API는 판매자용) **반자동**으로 설계한다:

```
재고 소진/임박 → 장보기 리스트 자동 추가
  + 소비 패턴 학습(Phase 2): "계란, 이번 주말쯤 떨어져요" 선제 알림
→ 리스트 화면: 품목별 [쿠팡에서 보기] 딥링크 (파트너스 태그)
→ 사용자: 탭 → 쿠팡 앱 장바구니 → 구매 (커미션 ~3% 발생)
→ 구매 후: 주문내역 캡처 입고로 루프 복귀
```

사용자 체감: "알림 받고 두 번 탭하면 장바구니" ≈ 자동에 가까운 경험. 우리: 주문 API 없이 장보기 전환마다 수익.
