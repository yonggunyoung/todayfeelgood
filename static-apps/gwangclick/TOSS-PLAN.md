## 0. 한눈 요약 (결론부터)
- **무엇**: '오늘의 떡밥(민초·부먹·아샷추…)'에 편 골라 60초 광클 → 전세계 실시간 진영전 + 친구 프라이빗 방. 10대~20대 캐주얼 밈 게임.
- **입점 형태(D-T1)**: 웹앱을 **앱인토스 web-framework로 빌드 → 콘솔 업로드**. (임의 URL 그대로가 아님.) 빠른 1차 검증은 **WebView 경로** 병행 검토.
- **광고 전략(D-T2)**: **1차는 광고 OFF로 출시**(사업자 등록 불필요·빠른 런칭) → 사업자 등록 후 **보상형 광고 ON**. 코드는 이미 `AD.enabled` 토글로 준비됨.
- **검수**: 4단계(운영·기능·디자인·보안), 평균 **2~3일**.
- **현재 완성도**: 게임·SDK 어댑터(`toss.js`)·아이콘·개인정보처리방침·스토어 문구·홍보팩 **준비 완료**. 남은 건 **web-framework 포팅 + 콘솔 등록 + (광고 시) 사업자 등록**.

## 1. 제품 개요 · 왜 토스에 맞나
- **장르**: 실시간 진영 클리커(탭) + 소셜 밈.
- **핵심 루프**: 떡밥 선택 → 60초 광클 → 전세계 집계 게이지·랭킹·승자 발표 → 다시.
- **소셜**: 코드/링크로 **친구·가족·학교 대항 프라이빗 방**(가입 불필요).
- **타깃**: 10~20대(토스 사용자층과 정합), 가벼운 도파민·바이럴(짧은 판·결과 공유 이미지).
- **토스 적합성**:
  - 30초~60초 단발 → 토스 안 "잠깐 들어와 노는" 미니앱에 최적.
  - 로그인/설치 마찰 0(익명+닉네임) → 전환률 높음.
  - 결과 이미지 공유·친구 방 → 토스 내/외 바이럴.
  - 비과금 기본, 수익화는 보상형 광고(옵트인)만 → 정책 충돌 적음.

## 2. 입점 형태 결정 (D-T1) — 중요
앱인토스 웹앱은 **두 경로**가 있다.

| 경로 | 내용 | 장점 | 단점 | 적합 |
|---|---|---|---|---|
| **A. web-framework 빌드** | 기존 웹에 `@apps-in-toss/web-framework` 설치 → 샌드박스 개발 → **빌드 산출물 업로드** | SDK(광고·햅틱·로그인·결제) 풀활용, 정식 권장 | 빌드 파이프라인·포팅 필요 | **정식 출시(권장)** |
| **B. WebView** | 기존 웹페이지를 토스 WebView로 표시 | 우리 `ddukkit.com/gwangclick` 거의 그대로 빠르게 | 네이티브 SDK 일부 제한·심사 기준 별도 | 빠른 1차 검증 |

**권장 순서**: ① WebView로 빠르게 동작/심사 감 잡기(선택) → ② **web-framework로 정식 빌드해 제출**.
> 현재 코드는 **둘 다 대비**돼 있음: `toss.js`가 SDK 전역(`window.AppsInToss`/`GoogleAdMob`/`getOperationalEnvironment`)을 **폴백 탐색**으로 흡수 → web-framework로 감싸도, WebView로 띄워도 동작.

### A 경로 포팅 작업(예상)
1. 새 앱인토스 web-framework 프로젝트 생성(그들의 CLI/템플릿).
2. 우리 정적 자산(`index.html`·`*.js`·`vendor/`)을 그 프로젝트의 웹 엔트리로 이식(번들러에 맞게 정리).
3. `toss.js`의 어댑터를 web-framework 실제 import(`@apps-in-toss/web-framework`의 `GoogleAdMob.loadAppsInTossAdMob`/`showAppsInTossAdMob` 등)로 1:1 연결(현재는 전역 폴백).
4. 샌드박스 앱으로 실기 테스트 → 빌드 업로드.

## 3. 입점 절차 & 일정 (공식)
**검수 4단계 / 평균 2~3일**
1. **운영 검수** — 앱 정보·노출 정보·서류 확인
2. **기능 검수** — 토스앱 안에서 오류 없이 동작
3. **디자인 검수** — 토스 UI 가이드 준수
4. **보안 검수** — 개인정보·보안 점검

> 콘솔에서 **개발 완료 전에도 앱을 미리 등록** 가능(메타 먼저 채워두기). 사업자 서류 심사는 별도 1~2영업일.

## 5. SDK 연동 계획 (현재 코드 상태 포함)
| 기능 | 토스 SDK | 사업자 등록 | 현재 코드 | 비고 |
|---|---|---|---|---|
| **보상형 광고** | `GoogleAdMob.loadAppsInTossAdMob`→`showAppsInTossAdMob`(adGroupId) | **필요** | `toss.js`+`AD.tossAdGroupId`(현재 OFF) | 결과화면 '2배'만, 옵트인·비개인화 |
| **햅틱** | 진동 피드백 | 불필요 | `Toss.haptic()`(밖에선 navigator.vibrate) | 탭 손맛 |
| **환경 감지** | `getOperationalEnvironment()` | 불필요 | `Toss.inToss()/env()` | toss/sandbox 분기 |
| **토스 로그인** | Toss Login | **필요** | 추상화 준비(현재 익명+닉/구글) | 입점 후 토스 로그인으로 교체 가능 |
| **인앱결제** | IAP | **필요** | 없음(무료) | 도입 시 15%+5% |

**연령 보호(D-T5)**: 보상형(옵트인)만, **비개인화(non-personalized)**, 배너/전면 없음 — 코드에 강제(`AD.nonPersonalized=true`).

## 6. 사업자 / 정산 / 수수료
- **사업자 없이 출시 가능.** 단 **인앱광고·토스로그인·토스페이·인앱결제·프로모션·비즈월렛**은 **사업자 등록 + 계약/정산** 필요.
- **광고 수익**: 광고 송출비 + 앱인토스 수수료 차감(대략 **운영비 ~30% 차감** 후 정산).
- **인앱결제 수수료**: 앱마켓 15% + 토스 5%(향후 변동 가능).
- → **결정(D-T2)**: 1차 광고 OFF로 빠르게 출시 → 트래픽 보고 사업자 등록 후 광고 ON.

## 7. 정책 · 심사 대비 (토스 가이드)
- **앱 내 완결**: 핵심 기능을 토스 밖으로 내보내 처리하지 않기. → 결과 **공유는 OK**지만 "외부 다운로드로만 완성" 금지(우리 게임은 앱 내 완결 ✅).
- **외부 유도 금지**: 자사앱 설치 유도/외부 링크 유도 금지 → `toss.js`의 `isToss()`로 **토스 안에선 "더 많은 앱(ddukkit.com)" 링크 숨김** 처리됨 ✅.
- **개인정보**: 최소수집·고지. `privacy.html`에 Firebase 익명집계·광고식별자·국기 CDN 고지 완료 ✅.
- **10대 보호**: 비개인화·옵트인 광고, 배너/전면 없음 ✅.
- **AI 표기**: 현재 AI 미사용 → 해당 없음(추후 AI 도입 시 "AI 생성" 라벨 의무).
- **디자인 가이드**: 토스 UI 톤(과한 광고·다크패턴 금지). 풀스크린·safe-area 처리됨 ✅.

## 8. 로드맵 (단계별)
**Phase A — 광고 없이 정식 출시 (빠름)**
1. 콘솔 워크스페이스 + 앱 메타 등록(이름/아이콘/스샷/설명/연령/개인정보URL)
2. web-framework로 빌드(또는 WebView) → 샌드박스 테스트
3. 빌드 업로드 → 4단계 검수(2~3일) → 출시
4. 스레드 등 홍보(`promo/`)

**Phase B — 사업자 등록 + 수익화**
5. 사업자 등록·계약·정산정보
6. 콘솔에서 **보상형 광고 그룹 생성** → `AD.tossAdGroupId`에 입력 + `AD.enabled=true`
7. (선택) 토스 로그인으로 신원 교체, 프로모션

**Phase C — 고도화**
8. 토스 사용자 데이터 기반 떡밥 큐레이션, 시즌/리그, 인앱결제(꾸미기) 검토

## 10. 리스크 & 결정 로그 (D-IDs)
- **D-T1 입점 형태**: web-framework 빌드 업로드(권장). 이유: SDK 풀활용·정식 경로. 비용: 포팅. 탈출구: WebView 1차.

- **D-T5 연령**: 비개인화·옵트인 보상형만, 배너/전면 0.
- **리스크**: ① web-framework 포팅 공수(번들 정리) ② 보상형은 사업자 필요(수익은 Phase B) ③ 토스 디자인 검수(과한 연출 지양 — 불꽃/연출 강도 조절 여지) ④ 도메인/배포 이원화(현 Oracle vs 토스 빌드) — 토스용은 별도 빌드 산출물.
