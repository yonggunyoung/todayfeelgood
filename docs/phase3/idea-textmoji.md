# 텍스트 이모티콘(textmoji) 집중 — 가치 게이트 + 조합엔진 설계

> 작성: 제품 연구 + 🔪 회의적 실사용자(가치 게이트). 코드 미수정. 검토: skeptic-product 7관문, roles.md, phase2/plan, idea-character-emoji, home/page.tsx, apps/sticker(presets/render 데이터 구조), apps/sign(SignStudio), packages/core 계약.
> 칭찬 없음. "그냥 검색해서 복붙하면 되지" 를 정면으로 다룬다.

---

## 한 줄 결론

**집중할 가치 "조건부 있음" — 단, 지금 마스터가 그린 그림 그대로면 또 '복붙 리스트 클론'으로 끝난다.** "ʕ•ᴥ•ʔ 같은 텍스트 이모티콘을 카테고리로 보여주고 원탭 복사" 만으로는 게이트를 못 넘는다. 인터넷에 이미 수천 개 복붙 리스트가 있고, 우리가 그걸 예쁘게 재포장한 N+1번째가 된다. **유일하게 게이트를 넘는 코어는 "절차적 조합 생성기" 단 하나다 — 부품(눈/입/팔/괄호/장식)을 조합해 *세상에 없던* 텍스트 이모티콘을 무한히 만들고, 그중 깨지지 않는 것만 안전등급으로 거른다.** 큐레이션 리스트는 이 생성기의 시드/부트스트랩일 뿐, 그 자체가 제품이면 안 된다. **1순위 빌드 = 조합 생성 엔진 + 호환성 안전등급 필터. 단순 복붙 갤러리는 게이트 ❌.**

## 1. 가치 게이트 판정 (7관문) — 봐주지 않음

핵심 질문 둘로 나눠서 판정한다.
- **(가) 단순 "큐레이션 리스트 + 원탭 복사"** = 인터넷 복붙 사이트의 깔끔한 클론
- **(나) "절차적 조합 생성기"** = 부품 조합으로 무한 생성 + 안전등급

| # | 관문 | (가) 큐레이션 리스트 | (나) 조합 생성기 |
|---|------|---------------------|------------------|
| G1 독자성(moat) | **❌** 후발이 주말에 복제. CSV 한 장 + `navigator.clipboard`면 끝. 데이터(이모티콘 목록)는 공개·자유복제라 데이터 우위도 없음 | **⚠️→PASS 가능** 부품 라이브러리 + 배치/조합 규칙 + **"기기별 호환성 등급 측정 데이터"**가 진짜 자산. 흉내는 가능하나 "안전등급이 신뢰할 만큼 정확"해지려면 누적이 필요 → 시간 해자 |
| G2 유용성 | △ 쓰긴 함(복사해서 카톡). 하지만 **이미 키보드·메신저·검색이 더 빠름** | **PASS** "남이 안 쓰는 *나만의* 텍스트 이모티콘"을 즉석 생성 → 차별 표현 욕구 충족. 10대의 "유니크함" 동기와 정합 |
| G3 완주 동기 | **PASS** 노동 0(보고 탭). 이건 (가)의 유일한 강점 | **PASS** 노동 0~경량. "랜덤 더보기" 한 탭이면 결과. 그리기 같은 숙제 없음. **손글씨앱의 26자 노동 문제가 여기엔 없다** = 구조적 우위 |
| G4 대체재 우위 | **❌ 치명** "그냥 검색해서 복붙" 이 더 빠르고 공짜. 아래 §1.1에서 정면 반박 시도했으나 (가)만으론 못 이김 | **⚠️→PASS** 대체재(복붙 리스트)는 *고정 목록*. 우리는 *생성*. "리스트에 없는 걸 만든다"가 복붙으로 안 되는 유일한 것 |
| G5 이탈 제거 | 이탈 이유: 검색이 빠름 / 깨져서 헛수고 / 볼 게 그게 그거 — **(가)는 1·3번 미해결** | 깨짐 = **안전등급으로 해결(핵심)**. 식상함 = **무한 생성·랜덤으로 해결**. 검색 빠름 = **"검색엔 없는 것"으로 무력화** |
| G6 결제 순간 | 없음. 복붙 리스트에 돈 낼 사람 0 | **약함(솔직히)** §7 참조. 무료·바이럴이 본령. 테마팩/시드공유가 약한 후보 |
| G7 안 만들 이유 | "인터넷에 널렸다" = 강력한 안 만들 이유. (가)는 **반박 실패** | "생성·안전등급·나만의 것" 세 축으로 반박 성공(§1.1) |

**판정 규칙(skeptic 문서): G1·G2·G3 중 하나라도 ❌ = 빌드 금지.**
- **(가) 큐레이션 리스트 단독: G1 ❌, G4 ❌ → 빌드 금지.** 이걸 메인으로 하면 안 된다.
- **(나) 조합 생성기: G1~G5 PASS(조건부), G6 약함 → 빌드 GO.** 단 G1 PASS는 "안전등급을 진짜로 정직·정확하게 만든다"가 전제. 그게 가짜면 다시 ❌.

### 1.1 "그냥 검색해서 복붙하면 되잖아?" — 정면 반박 / 일부 인정

**인정할 것(거짓말 안 함):**
- *이미 존재하는* 흔한 이모티콘(ㅋㅋ, (T_T), ¯\\\_(ツ)\_/¯, ☞☜)은 **검색·복붙이 우리보다 빠르다.** 이 영역에서 우리는 못 이긴다. "예쁜 갤러리"는 마케팅이지 가치가 아니다.
- 텍스트 이모티콘은 **재방문 동기가 약하다.** 한 번 자주 쓰는 5개를 외우면 앱을 다시 안 연다. (가)는 이 약점에 그대로 노출.

**반박할 것(우리가 나은 단 하나):**
- 복붙 리스트는 **고정된 유한 목록**이다. "ʕ•ᴥ•ʔ 인데 *내 캐릭터처럼* 모자 씌우고 하트 들고 윙크하는 버전"은 **검색해도 안 나온다.** 그건 *생성*해야 한다.
- 따라서 우리의 존재 이유 = **"검색으로 찾을 수 없는, 방금 만들어진 조합"**. 큐레이션은 누구나 하지만, **신뢰할 수 있는 절차적 생성 + 깨짐 안전등급**은 복붙 사이트가 안 한다.
- 부차적으로 **호환성 보증**: 복붙 사이트는 "이거 안드로이드에서 깨짐"을 안 알려준다. 붙여넣고 나서야 안다. **우리는 붙여넣기 전에 등급으로 경고** → 헛수고 제거(G5).

**결론:** "복붙하면 되잖아"는 **(가)에 대해선 맞고, (나)에 대해선 틀리다.** 그러므로 (나)만 만든다. 큐레이션 리스트는 (나)의 시드 데이터로 흡수.

## 2. 싸인(sign) 처리 권고 — **구석 보류(유지하되 비활성 톤), 제거 아님**

- **완전 제거 ❌.** 코드(`apps/sign`, SignStudio, 절차적 장식 엔진)가 이미 동작하고, "문서·이메일 서명" 이라는 *어른* 사용처가 실재한다. 지우면 그 소수 사용자와 작업물을 버린다. 매몰비용 회수 차원이 아니라 **타깃이 다른 보조 도구**로 살아있을 가치는 있다.
- **다른 용도 전환 △(약함).** 싸인 엔진(흘림체 변형 + 절차적 장식)을 textmoji로 전용? 메커니즘이 다르다(벡터 글리프 vs 순수 유니코드 문자열). 재활용 가치 낮음. 전환 권하지 않음.
- **권고: home에서 "구석"으로 강등 + 타깃 라벨 정직화.** 근거:
  1. 마스터 진단대로 **10대는 싸인 쓸 일이 거의 없다.** 메인 동선에서 빼는 건 맞다.
  2. 단 "직장인·문서" 타깃에겐 유효 → **삭제가 아니라 "더 보기/도구 전체" 영역으로 이동.**
  3. home 카드 4개(폰트·스티커·싸인·키트)를 **"textmoji 1개 전면 + 나머지 4개를 접힌 '다른 도구' 영역"** 으로 재배치(§5.3).
- **한 줄: 싸인 = 죽이지 말고 구석에. textmoji가 무대 중앙.**

## 3. "엄청나게 다양하게" 조합 생성 엔진 설계 (핵심)

> 기존 `apps/sticker/lib/presets.ts`가 이미 **부품(eyes/mouth/deco)을 데이터 배열로 두고 조합**하는 패턴을 증명했다. textmoji는 같은 사상을 **그래픽 합성이 아니라 유니코드 문자열 조합**으로 옮긴 것. 서버 0, 비용 0, 클라에서 문자열 조립.

### 3.1 부품 라이브러리 (전부 데이터 = 코드 수정 없이 트렌드 추가)

텍스트 이모티콘 = `[왼팔] [왼괄호] [왼눈] [입/코] [오눈] [오른괄호] [오른팔]` 슬롯에 부품을 끼우는 것. 각 슬롯 = 후보 문자 배열.

| 슬롯 | 예시 부품(문자) | 비고 |
|------|----------------|------|
| eyes(눈) | `•` `◕` `^` `>` `T` `╥` `ᵔ` `°` `⊙` `￣` `＾` `๑` `≧` `╹` | 감정의 핵심. 좌우 같은/다른 눈 조합 |
| mouth(입/코) | `ᴥ` `ω` `ᗢ` `‿` `╭╮` `Д` `□` `o` `﹏` `▽` `^` `ε` | 동물상(ᴥ)/사람상(‿) 분기 |
| cheeks/face(볼·윤곽) | `(` `)` `｡` `（` `）` `〔` `〕` `ʕ` `ʔ` `╰` `╯` | ʕʔ=곰, ()=기본 |
| arms(팔) | `╯` `╰` `┛` `ノ` `つ` `o` `ヽ` `୧` `୨` `/` `\\` | 던지기(╯°□°）╯, 안기(づ) |
| hands/items(손·소품) | `︵` `┻━┻` `♡` `☆` `✿` `≡` `⌐■` | 책상뒤집기·하트던지기 등 액션 |
| deco(장식) | `✧` `｡ﾟ` `♡` `★` `~` `…` `!!` `♪` | 분위기 강화 |

**조합 규칙(배치):** 슬롯 채움 순서 + 대칭 규칙(왼눈↔오른눈 미러: `◕`→`◕`, `>`→`<`, `╯`→`╰`)으로 자연스러운 좌우 대칭/액션형 생성. 미러 매핑 테이블도 데이터.

### 3.2 절차적 생성 = 부품 × 감정 × 스타일

```
생성(emotion, style, seed):
  부품풀 = 감정별 가중 후보(joy → eyes∈{^,◕,ᵔ}, mouth∈{‿,ᴥ,▽}, deco∈{✧,♪})
  rng = mulberry32(seed)          # apps/sticker/lib/rng.ts 재사용 패턴
  눈 = rng.pick(부품풀.eyes)
  입 = rng.pick(부품풀.mouth)
  얼굴괄호 = style.faceBrackets ? rng.pick(brackets) : ["",""]
  팔/소품 = style.action ? rng.pick(actions) : null
  문자열 = assemble(슬롯들, 미러규칙)
  return { text, seed, parts, safety: rate(문자열) }
```
- **시드 재현:** 같은 (emotion, style, seed) → 항상 같은 결과. 공유 URL `?e=joy&s=cute&seed=4821` 로 "내가 만든 그거" 재현(스티커앱의 시드 갤러리 사상 동일).
- **"랜덤 더보기"(의외성):** seed만 증분/난수 → 9~12개 그리드를 매 탭 새로. 손글씨앱이 못 가진 **노동 0의 무한 다양성**이 여기 본령.
- **감정 카테고리(시드):** 기쁨·슬픔·사랑·화남·놀람·귀여움·시크·당황·뿌듯·민망·졸림·하트·인사·OK·거절·축하 — **데이터 배열**. 트렌드 감정(예: "킹받음", "무야호")도 항목 추가만으로.
- **스타일 축:** `동물상/사람상`, `심플/화려(deco량)`, `정적/액션형`, `대칭/비대칭`. 슬라이더가 아니라 **칩 토글**(모바일 친화).

### 3.3 큐레이션 + 생성 혼합 (품질 관리 = 필수)

순수 랜덤 조합은 **상당수가 못생기거나 안 읽힌다.** 그래서:
- **큐레이션 시드 세트:** 검증된 "명작" 100~300개를 `curated[]` 데이터로 보유(공개 리스트에서 수집, 라이선스 무관한 일반 조합). 카테고리 첫 화면 = 큐레이션 우선 노출.
- **생성은 "더 만들기/랜덤" 에서:** 사용자가 명시적으로 "변형·랜덤" 누를 때만 절차 생성. 첫인상은 항상 검증된 것.
- **즐겨찾기 = 사용자 큐레이션:** 사용자가 별 누른 것 로컬 저장(localStorage, 서버 0). 추후 "인기 시드" 집계로 큐레이션 보강(데이터 자산 = G1 강화).
- **하드 필터:** 생성물이 안전등급 "위험"이거나 슬롯 충돌(괄호 불균형 등)이면 자동 폐기 후 재생성.

### 3.4 호환성 정직 (skeptic 정직성 원칙 — 절대 타협 금지)

**이게 우리의 진짜 차별이자 가장 거짓말하기 쉬운 부분.** 정직하지 않으면 G1·G5가 무너진다.

- **문제:** 결합문자(combining), 일부 ʕ ᴥ ◕ 같은 확장 라틴/IPA/한정 유니코드는 **기기·앱·폰트마다 두부(□)·물음표·깨짐.** 안드로이드 구버전, 카톡 vs 인스타 vs 디스코드가 제각각.
- **안전등급(데이터로 관리):**
  - 🟢 **안전**: ASCII + 광범위 지원 BMP(예: `^_^` `T_T` `:3` `(>_<)` 기본 괄호/별표). 어디서나 100%.
  - 🟡 **보통**: 흔한 확장(ᴥ ◕ ω ノ) — 최신 기기 OK, 구형/일부 앱 위험.
  - 🔴 **화려/위험**: 결합문자·희귀 블록·이모지혼합 — 깨질 확률 높음. "내 폰에선 예뻐도 상대 폰에선 □" 경고.
- **측정 방식(정직하게):** 처음엔 **유니코드 블록·버전 기반 휴리스틱**으로 등급 자동 산정(combining/Unicode≥some version/희귀 블록 → 낮은 등급). 완벽하지 않음을 UI에 명시("추정치"). 추후 사용자 신고("깨져요" 버튼)로 등급 보정 → **누적 호환성 데이터 = 복붙 사이트가 절대 안 하는 자산(G1).**
- **UI 규칙:** 각 항목에 등급 배지. 복사 시 🔴면 "상대 기기에서 깨질 수 있어요" 토스트. **거짓 안심 금지.**

## 4. 10대 실사용 시나리오 — 무엇이 "한 번 더 쓰고 공유"하게 하나

- **카톡/DM:** 친구 메시지에 반응. 길게 치기 귀찮은 감정을 한 글자처럼. **반응 세트**(같은 감정 5종)를 연타로 도배 = 10대 화법. → **"감정별 5종 묶음 한 번에 복사"** 기능이 핵심.
- **인스타 스토리/프로필/소개:** 닉네임 꾸미기(`˗ˏˋ name ´�î˗`), 스토리 텍스트 데코. 여기선 **🟢안전등급 + 좌우대칭 장식형**이 주력.
- **디스코드:** 채팅 도배·역할명·상태. 모노스페이스 환경이라 ASCII 아트형이 잘 먹음 → 디스코드 프리셋(🟢 위주).
- **"한 번 더 쓰고 공유"를 만드는 것:**
  1. **원탭 복사** — 마찰 0. 본 즉시 손가락 하나.
  2. **반응 세트** — 하나가 아니라 "이 감정 한 묶음" → 대화에서 연속 사용.
  3. **나만의 것(생성)** — "이거 내가 만든 거임" 자랑 → 공유 URL(시드) → 친구도 앱 유입(바이럴 루프).
  4. **트렌드 감정 칩** — 요즘 밈 감정("킹받음")이 최상단 → 시의성으로 재방문.

## 5. UI 집중 설계 (모바일 우선, 단일 목적)

### 5.1 단일 화면 구성(스크롤 없이 핵심 — plan UX 원칙 준수)
```
┌──────────────────────────────┐
│ [검색바: 감정·키워드]   [🎲랜덤]│  ← 상단 고정(sticky)
│ [감정 칩: 기쁨 슬픔 사랑 시크…→]│  ← 가로 스크롤 칩(트렌드 좌측 우선)
├──────────────────────────────┤
│  ʕ•ᴥ•ʔ   🟢  [복사] [★]       │  ← 카드 그리드(2~3열)
│  (｡•́‿•̀｡) 🟡  [복사] [★]       │     각 카드: 미리보기·등급·원탭복사·즐겨찾기
│  ◕‿◕     🟢  [복사] [★]       │
│  …                            │
│  [+ 이 감정 더 만들기(랜덤)]   │  ← 절차 생성 진입
├──────────────────────────────┤
│ [전체] [즐겨찾기★] [내가만든것]│  ← 하단 탭바(모바일)
└──────────────────────────────┘
```
- **원탭 복사:** 카드 탭 = 즉시 클립보드 + "복사됨" 마이크로 피드백(너굴이 "받았다 너굴"). 모달·페이지 이동 0.
- **검색:** 감정 라벨·키워드(곰/하트/우는/책상)·부품 태그로. 한국어 우선.
- **랜덤(🎲):** 의외성 후크. 매 탭 새 그리드.
- **즐겨찾기:** localStorage, 별 토글. 서버·로그인 0.
- **등급 배지:** 모든 카드에 🟢🟡🔴. 탭당 정직성.
- **반응 세트:** 감정 칩 길게 누르면 "이 감정 5종 한 번에 복사".

### 5.2 모바일 우선 근거
- 10대 = 모바일. 텍스트 이모티콘 = 메신저 = 모바일. **PC는 부차.**
- 터치 타깃 큼, 가로 칩 스크롤, 하단 탭바. plan의 "한 시야·sticky·반응형 일관" 그대로.

### 5.3 home 재배치 (나머지 앱 "구석")
- **현재:** 폰트·스티커·싸인·키트 4카드 균등.
- **변경안:** 히어로 CTA = **textmoji 전면**("지금 복사해서 카톡에 써 너굴"). 그 아래 **"다른 공방" 접힘 영역**에 폰트·스티커·싸인·키트 작은 카드. 싸인은 그 안에서도 뒤쪽.
- 자기잠식 주의(home page.tsx 주석): textmoji 핵심 키워드("텍스트 이모티콘", "감정 이모티콘")는 `/textmoji` 랜딩에 집중, home은 허브 일반어 유지.

## 6. 앱 구조 — **신규 `apps/textmoji` 권고**

- **신규 vs 기존:** 신규 `apps/textmoji/frontend` 권고.
  - 기존 `apps/sticker`는 **그래픽 PNG 합성**(Canvas·이미지). textmoji는 **순수 유니코드 문자열**. 입력·출력·사용맥락이 다르다 → 한 앱에 섞으면 UX·토스 심사 모호(skeptic·idea-character-emoji가 sticker를 별도 앱으로 분리한 동일 논리).
  - **서버 불필요(서버 0):** 문자열 조립·복사·즐겨찾기 전부 클라. `engine` 폴더 불요(sticker처럼 frontend만). 무료티어 부담 0.
- **공용 재사용(복붙 금지, CLAUDE.md):**
  - `packages/core`: textmoji 계약 추가(§8). 단일 출처.
  - 신규 `packages/textmoji-kit`(또는 core 하위): 부품 라이브러리·감정/스타일 데이터·미러 매핑·안전등급 휴리스틱·큐레이션 시드. **데이터는 여기 한 곳** → 트렌드 추가 = 이 패키지만.
  - `packages/ui`: 칩·세그먼티드·너굴이·카드 재사용.
  - `apps/sticker/lib/rng.ts`(mulberry32 시드)·`presets.ts`의 데이터배열 패턴을 textmoji-kit가 동일 채택.
- **토스 대비:** 앱 내 완결(복사=인앱), 외부 다운로드 강제 없음, 광고망 직접삽입 없음. 생성물 정직 라벨(절차 생성). 결제/로그인은 추상화 자리만(현재 미사용).
- **home·infra:** home에 textmoji 전면 카드, `infra/nginx`에 `메인도메인/textmoji` 라우팅 추가(CLAUDE.md 새 앱 규칙).
- **이름:** `textmoji`(검색·SEO 명료). 한국어 노출명 "이모티콘공방" 또는 "텍스트이모티콘".

## 7. 수익화/그로스 — 솔직히 "성장 우선"

- **무료·바이럴이 본령.** 텍스트 이모티콘에 돈 낼 사람은 거의 없다. 거짓말 안 함(G6 약함).
- **그로스(진짜 동력):**
  - **원탭 복사 + 공유 URL(시드)** = 마찰 0 바이럴. "내가 만든 거 봐" → 친구 유입.
  - **OG 공유 페이지** `/textmoji/s/[seed]`: 시드로 결과 재현 + OG 썸네일 → 카톡 링크 미리보기 → SEO/유입(sticker의 OG 루프 재사용).
  - **트렌드 감정 칩**으로 재방문(시의성).
- **유료 여지(약함, 무리 금지):**
  - **테마팩/프리미엄 부품:** "시즌 한정 부품(크리스마스·할로윈)", "장식 폰 라이브러리 확장". 가능은 하나 모수 작음.
  - **광고 없는/대량 즐겨찾기 동기화** — 로그인 필요, 시기상조.
  - **솔직한 권고: Phase 3에선 무료 100% + 바이럴.** 결제는 사용자·트래픽 생긴 뒤. G6 약점을 인정하고 "성장으로 보완"이 정답. 결제를 코어에 억지로 박지 말 것.

## 8. 계약/데이터 모델 (트렌드 = 데이터 추가만으로)

`packages/core`에 하위호환으로 추가(기존 FontParams/Sticker 계약 불변). 실제 부품/큐레이션 데이터는 `packages/textmoji-kit`.

```ts
// ── 텍스트 이모티콘(textmoji) 계약 ──

/** 호환성 안전등급(정직성). heuristic 추정 + 사용자 신고 보정. */
export type SafetyTier = "safe" | "ok" | "fancy"; // 🟢 / 🟡 / 🔴

/** 부품 슬롯 — 데이터로 채우는 조립 자리 */
export type PartSlot = "face" | "eyeL" | "eyeR" | "mouth" | "armL" | "armR" | "item" | "deco";

/** 한 부품 = 문자 + 미러 짝(좌우 대칭 생성용) + 등급 힌트 */
export interface TextPart {
  id: string;
  slot: PartSlot;
  glyph: string;          // 실제 유니코드 문자(열)
  mirror?: string;        // 좌우 미러 시 대체 글자(예: ">"→"<", "╯"→"╰")
  tier: SafetyTier;       // 이 부품 자체의 등급(전체는 max로 합산)
  tags?: string[];        // 검색·감정 매칭("곰","우는","하트")
}

/** 감정 카테고리 — 데이터 배열(트렌드 추가 = 항목만) */
export interface EmotionCat {
  id: string;             // "joy" | "sad" | "king-bad" ...
  label: string;          // "기쁨" "킹받음"
  emoji: string;          // 칩 아이콘
  trend?: boolean;        // 트렌드면 칩 좌측 우선
  // 슬롯별 선호 부품 id 풀(가중 랜덤)
  prefer: Partial<Record<PartSlot, string[]>>;
}

/** 생성 스타일 축(칩 토글) */
export interface TextmojiStyle {
  animalFace: boolean;    // 동물상(ᴥ) vs 사람상(‿)
  fancy: number;          // deco량 0~2(심플~화려)
  action: boolean;        // 팔/소품 액션형
  symmetric: boolean;     // 좌우대칭
}

/** 절차 생성 요청(클라 내부 — 서버 0) */
export interface TextmojiGenRequest {
  emotion: string;        // EmotionCat.id
  style: TextmojiStyle;
  seed: number;           // 재현·공유
  maxTier?: SafetyTier;   // "safe만 보기" 필터
}

/** 생성/큐레이션 공통 결과 항목 */
export interface TextmojiItem {
  text: string;           // 완성된 텍스트 이모티콘
  tier: SafetyTier;       // 합산 등급(부품 max + 결합문자 검사)
  seed?: number;          // 생성물이면 재현용
  emotion?: string;
  source: "curated" | "generated"; // 정직성 라벨
  parts?: string[];       // 사용 부품 id(디버그·리믹스)
}

/** 큐레이션 시드(데이터) — 검증된 명작 */
export interface CuratedTextmoji {
  text: string;
  emotion: string;
  tier: SafetyTier;
  tags?: string[];
}

/** 안전등급 휴리스틱(추정치 — UI에 "추정" 명시) */
export function estimateTier(text: string): SafetyTier {
  // combining mark / 희귀 블록 / Unicode 버전 높음 → 낮은 등급
  // ASCII+흔한 BMP → "safe". (실제 구현은 textmoji-kit)
  return "ok";
}

export const MAX_TEXTMOJI_SET = 12; // 한 화면 그리드(스티커앱 사상 동일)
```
- **트렌드 반영:** 새 감정/부품/큐레이션은 **데이터 배열에 항목 추가만** → 코드 수정 0(plan "데이터로 쉽게 추가" 원칙).
- **정직성:** `source`("curated"|"generated") + `tier` 항상 포함 → UI 강제 노출(스티커앱 `generatedBy`·등급 패턴 동일).

## 우선순위

1. **[1순위·MUST] 조합 생성 엔진 + 안전등급 휴리스틱** (`packages/textmoji-kit`). 부품 데이터 + 미러규칙 + 감정풀 + estimateTier. 이게 코어이자 유일한 moat. **단순 갤러리부터 만들면 안 됨(게이트 ❌).**
2. **[MUST] 큐레이션 시드 100~300개** + 카테고리 첫 화면(검증된 것 우선 노출). 생성은 "더 만들기/랜덤"에서.
3. **[MUST] 모바일 단일 화면 UI**: 검색·감정칩·카드그리드·원탭복사·🎲랜덤·★즐겨찾기·등급배지. 서버 0.
4. **[해야] 공유 URL(시드) + OG 페이지** `/textmoji/s/[seed]` — 바이럴 루프(스티커 OG 재사용).
5. **[해야] 사용자 "깨져요" 신고 → 등급 보정** 누적(G1 강화 데이터 자산). 초기엔 휴리스틱만으로 시작 가능.
6. **[home] textmoji 전면 + 싸인 포함 나머지 앱 "다른 공방" 접힘 영역으로 강등.**
7. **[보류] 수익화.** Phase 3 무료 100%. 테마팩/프리미엄 부품은 트래픽 확인 후.

**게이트 재확인:** 빌드는 **(나) 조합 생성기 경로로만** 착수. 만약 일정·역량상 "큐레이션 리스트만 먼저" 로 후퇴하면 **그건 게이트 ❌ 제품**(복붙 사이트 클론)이므로, 그 경우 출시하지 말고 생성 엔진을 먼저 완성할 것. 안전등급의 정직성이 무너지면 G1·G5도 함께 무너진다 — 거짓 안심 절대 금지.

## 요약 (마스터용)

- **판정:** 단순 "텍스트 이모티콘 큐레이션+복붙 갤러리"는 **게이트 ❌**(G1 독자성·G4 대체재 — "검색해서 복붙"이 더 빠름, 인터넷에 널림). **"절차적 조합 생성기 + 호환성 안전등급"** 만이 게이트 통과(G1~G5 조건부 PASS, G6 결제는 약함=성장 우선). "검색으로 못 찾는, 방금 만든 나만의 것"이 유일한 존재 이유.
- **1순위 빌드:** `packages/textmoji-kit` — 부품 라이브러리(데이터) × 감정 × 스타일을 시드로 절차 생성 + 안전등급(🟢🟡🔴) 휴리스틱. 그 위에 모바일 단일화면(검색·감정칩·원탭복사·랜덤·즐겨찾기·등급배지), 신규 `apps/textmoji`(서버 0, frontend만). 큐레이션 리스트는 생성기의 시드일 뿐, 단독 출시 금지.
- **싸인 권고:** **제거 아님 — 구석 보류.** 10대는 안 쓰니 home 메인에서 빼되, 문서·서명 타깃에겐 유효하므로 "다른 공방" 접힘 영역으로 강등. 코드·기능은 유지.
