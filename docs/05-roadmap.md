# 05. 로드맵 & KPI

## Phase 0 — 준비 (1~2주)

- [ ] 쿠팡 파트너스 가입 신청 (승인 리드타임 있음 — 가장 먼저)
- [ ] 식약처 레시피 DB 다운로드 → `recipes` 스키마로 가공 (재료 정규화 포함)
- [ ] `ingredients_master` 상위 ~500개 재료 큐레이션 (동의어, 기본 소비기한, qtyType, 영양)
- [ ] 산모 모드 금기·주의 식재료 목록 큐레이션 (출처 명기)
- [ ] 화면 와이어프레임 5장: 홈 / 입고 확인 / 레시피·조리 / 차감·잔반 시트 / 장보기
- [ ] 앱 이름 확정 + 상표 검색 (키프리스)

## Phase 1 — MVP (6~8주)

코어 루프 한 바퀴가 완전히 도는 것이 목표. 이 외의 것은 전부 쳐낸다.

| 주차 | 범위 |
|---|---|
| 1~2 | Expo 셋업, Auth, `ingredients_master`/`recipes` 시드 적재, 재고 CRUD(퀵추가) |
| 3~4 | 영수증 AI 파싱 파이프라인(Functions + 한도 미터링), 입고 확인 화면 |
| 5 | 추천 엔진(비AI) + 모드 2개(운동·자린고비), 홈 화면 |
| 6 | 조리 플로우: 요리 완료 차감 시트, 잔반 등록·타이머 |
| 7 | 장보기 리스트 + 쿠팡 파트너스 링크, 임박 푸시(FCM) |
| 8 | 구독 결제(IAP/웹), 온보딩(첫 영수증 무료 와우), 폴리싱 → **Android+웹 출시** |

MVP에서 의도적으로 뺀 것: 산모 모드(금기 목록 검수 후 Phase 2), 가족 공유, 유튜브, 커뮤니티, 바코드(퀵추가로 대체 가능).

## Phase 2 — 확장 (출시 후 1~3개월)

- 소비 패턴 학습 → "떨어질 때쯤" 선제 알림 (반자동 재주문 완성)
- 산모 모드 + 모드 추가 (다이어트, 이유식)
- 가족 공유 냉장고 (프리미엄 간판 기능)
- **유튜브 레시피 추출**: 링크/공유 → 구조화 레시피 → 내 재고와 매칭 ("이 영상 요리, 지금 우리 집 재료로 가능?") — 경쟁 영상레시피 앱들과의 정면 차별화 지점
- iOS 출시, 월간 정합성 체크(냉장고 사진 대조)

## Phase 3 — 플랫폼화 (3개월~)

- 커뮤니티: 레시피 공유, 모드별 식단 인증, 절약 챌린지
- 크리에이터 제휴: 요리 유튜버 공식 레시피 카드
- 제휴 확장: 컬리, KAMIS 가격 연동 "이번 주 싼 재료" 추천

## KPI (단계별 북극성)

| 단계 | 북극성 지표 | 보조 지표 |
|---|---|---|
| MVP | **주간 요리완료(차감) 이벤트 ≥ 2회/사용자** — 코어 루프가 도는지 | D7 리텐션 ≥ 25%, 입고→첫 추천 도달율 |
| 성장 | D30 리텐션 ≥ 15%, 장보기 링크 CTR | 잔반 소진율(먹음/버림 비율), AI 한도 도달률 25~40% |
| 수익 | 구독 전환율 ≥ 3~5%, 파트너스 월 GMV | 해지율 < 8%/월, AI 변동비/MAU |

해석 기준: "요리완료 이벤트"가 안 나오면 차감 UX 실패 = 제품 실패. 다른 모든 지표보다 먼저 본다.

## 리스크 레지스터

| 리스크 | 대응 |
|---|---|
| 입력 마찰로 재고 데이터 부패 (카테고리 공통 사망 원인) | 영수증 1장 입고 + 원탭 차감에 개발 자원 최우선 배분. 정합성 체크로 보정 |
| AI 파싱 정확도 (한국 영수증 축약명) | 출시 전 영수증 100장 벤치마크, 정규화 사전 지속 보강, 확인 화면으로 오류 흡수 |
| 쿠팡 파트너스 정책 변경/승인 거절 | 컬리 어필리에이트·링크프라이스 등 대체 채널 준비 |
| 무료 AI 한도 남용 (계정 어뷰징) | 디바이스+계정 결합 미터링, 일일 비용 상한 알람 |
| 산모 모드 법적 리스크 | Phase 2로 미루고 검수 후 출시, 고지문 + 하드 필터 이중화 |
| 1인 개발 번아웃 | MVP 범위 동결. "이것도 넣을까"는 전부 Phase 2 백로그로 |
