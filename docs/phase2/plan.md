# Phase 2 마스터플랜 (아이디어 6종 종합)

> 6개 아이디어 에이전트(한글/손글씨/스타일/수익화/캐릭터·이모지/디자인) 산출물을 종합한 실행 계획.
> 원칙: **비용0 비AI 우선, AI는 유료·기본 OFF, 정직성(무엇이 진짜 내 글씨인지 고지), 무료티어 메모리 의식.**

## 0. 한 줄 비전
"슬라이더로 노는 폰트"에서 → **내 글씨·한글·괴상함·무드까지 고르고, 스티커/캐릭터로도 뽑아내는** 창작 도구로. 무료로 입소문, 창작성·라이선스·시간에 과금.

## 1. 빌드 웨이브 (우선순위)

### ✅ Wave A — 디자인 전면 재설계 (지금)
"클로드 코드 냄새" 확실 제거. **"소프트 iOS 문방구"** 디자인 시스템 + 마스코트 "획이".
- 토큰: 바탕 `#f6f5f2`, 잉크 `#2b2a33`(완전검정 금지), 악센트 `#5b6cf0`, 캔디 파스텔 소량, 라운드 8~32px(연속곡률), 소프트 그림자/클레이/블러, 스프링 모션, Noto Sans KR(또는 Pretendard).
- 세리프·올캡스 트래킹·전면 1px 보더·균등 3단 스텝·`<br>` 히어로 전부 폐기.
- iOS 패턴: 라지타이틀, 세그먼티드 컨트롤, 카드 리스트, 시트, (모바일) 탭바. 홈+스튜디오 동기화.

### ✅ Wave B — 스타일 다양화 (지금, 거의 공짜)
기존 변형 엔진 재사용. 체감 가치 최상.
- **버그**: UI weight(100~900)를 베이스폰트 실제 wght 범위로 매핑(현재 가는굵기 미작동).
- **미사용 축 개방**: MONO(모노스페이스), CRSV(필기체 전환) 슬라이더 (라틴/Recursive 전용).
- **괴상함(weirdness) 0~100**: 시드 기반 펜 디스토션(jitter+baseline wobble), 주사위 버튼으로 재현.
- **자간(letterSpacing)**, **무드 프리셋 6종**(차분/장난기/타자기/기하/거친손글씨/우아한필기 — 프론트에서 축 조합).
- 정형화(클린)=「차분」프리셋+weirdness 0.

### ✅ Wave C — 한글 확장 (지금)
- **H0 즉시**: OFL 한글 가변폰트(Pretendard/Noto KR) 베이스로 **변형 엔진 재사용**. script 토글(latin/hangul). 한글엔 곡률/MONO/CRSV 없음 → 숨김. slant는 합성 shear, weirdness/자간은 펜으로 공통 적용.
- 서브셋: 상용 2,350자(KS X 1001) 우선(메모리). OFL 자체 패밀리명·라이선스 동봉.
- 정직성: "공개 폰트 변형(내 글씨 아님)" 고지.

### 🔜 Wave D — 내 글씨 심화 (다음)
- 비AI **글리프 직접 캡처**(그린 글자→Potrace 벡터화→fontTools 조립=진짜 내 글씨). Pillow/OpenCV/Potrace 도입(메모리 검토).
- quick win: **부분 스타일 추출**(그린 글씨 굵기/기울기/곡률 통계→기존 축 자동 매핑).

### 🔜 Wave E — 스티커/캐릭터 (다음, 신규 앱)
- `apps/sticker`: 한 번 그리면 표정/색 변주 N종 자동 → 투명 PNG/APNG 팩(비AI, 브라우저 합성). 토스 유사앱 중복 회피로 별도 앱.

### 🔜 Wave F — 수익화 + AI (이후)
- P0(무코스트): 워터마크 제거·상업 라이선스·풀포맷(TTF/OTF/웹폰트킷) 내보내기 유료.
- AI: 내 손글씨 완성(한글 포함) few-shot, 크레딧 과금, 기본 OFF, "AI 생성" 라벨. 결제/로그인 provider 추상화(토스 대비).

## 2. 이번 빌드(A+B+C) 분담
- **Engine-Agent**(Python): weight 매핑 버그, MONO/CRSV, weirdness 펜(시드), 자간, 한글 베이스폰트+script, 상용 서브셋, 정직성 메타. 계약 v3 반영.
- **FE-Agent**(Node): 새 디자인 시스템+마스코트(홈+스튜디오), 추가 컨트롤(슬라이더/프리셋칩/괴상함+주사위/script 세그먼티드), 접근성 유지. 계약 v3 반영.
- 계약(`packages/core`)은 오케스트레이터가 v3로 선반영(단일 출처).
- 이후 **Critique 반론 → 수정 → 실 e2e/스크린샷 검증.**

## 3. 정직성/정책 체크 (전 웨이브 공통)
- 결과물 출처 라벨: 공개폰트 변형 / 내 글씨 / AI 생성 명확히.
- AI 외부 API는 env on/off + 비용 경고 주석 + 기본 OFF.
- OFL 폰트 재배포 시 자체 패밀리명 + 라이선스 동봉.

## 4. UX 원칙 (전 화면 공통 — 마스터 지시)
- **한 시야(single viewport) 처리**: PC·모바일 모두 핵심 흐름(조절 → 프리뷰 → 받기)을 한 화면에서 끝내는 걸 기본으로. 스크롤 없이 의도 전달.
- **시의성(immediacy)**: 화면을 벗어나야 할 때도 **프리뷰와 핵심 액션(받기/포맷)이 항상 보이게** — sticky 프리뷰, 모바일 하단 시트/고정 바, 접근 가능한 위치 고정.
- **균형(balance)**: 컨트롤 과밀 금지. 빠른시작/세부/고급(아코디언) 위계로, 첫 화면은 가볍게. 데스크톱=프리뷰 고정 2열, 모바일=프리뷰 상단 고정+컨트롤 시트.
- **반응형 일관성**: 같은 기능이 두 폼팩터에서 동등하게 도달 가능. 터치 타깃·키보드 모두 고려.
