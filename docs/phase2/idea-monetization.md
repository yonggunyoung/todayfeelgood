# Phase 2 — 제품·수익화 아이디어 설계서

> 작성: 제품·수익화 아이디어 에이전트 / 코드 수정 없음, 설계 문서.
> 전제: 1인 운영 · Oracle 무료티어 · 비용 0 기본 · AI는 유료/기본 OFF · 향후 토스 미니앱.
> 현재 상태: `apps/font/`는 라틴(a–z/대문자/숫자) **가변폰트 파라미터 변형**(굵기/기울기/곡률) 생성기, WOFF/TTF **무료 다운로드**.

## 0. 한 줄 결론 + 1순위 추천

**한 줄 결론**: "변형 폰트 무료 다운로드"는 지금 그대로 SEO·입소문 미끼로 두되, 돈은 **(1) 내 손글씨 → 한글 포함 완성 폰트(AI)** 와 **(2) 상업용 라이선스 + 워터마크 제거 + TTF/OTF·웹폰트킷 내보내기**에서 받는다.

**1순위 유료 기능**: **"내 손글씨 완성 폰트(한글 포함) + 상업용 라이선스 다운로드"**.
- 이유: ① 지갑을 여는 진짜 동기는 "변형된 기존 폰트"가 아니라 **세상에 없던 내 글씨를 폰트로 소장/상업 사용**하는 것. ② 한글 손글씨 폰트는 글자 수(2,350~11,172자)가 많아 수작업 비용이 크고 시장 단가도 높아 **결제 저항이 가장 낮은 구간**. ③ AI 원가를 1회성 생성 과금으로 그대로 전가 가능 → 1인 운영 적자 리스크 최소.
- 단, AI 엔진 완성 전까지의 **선행 수익화**는 "상업용 라이선스 + 워터마크 제거 + TTF/OTF 내보내기"로 먼저 시작(이미 가진 변형 엔진만으로 매출 가능).

## 1. 유료가 정당화되는 기능 발굴 (가치 근거 포함)

| 기능 | 무엇 | 사람이 지갑을 여는 이유(가치 근거) | 원가/난이도 |
|------|------|--------------------------------|------------|
| **내 손글씨 완성 폰트(AI)** | 글자 몇 개 입력 → 전체 글리프 AI 생성 | "세상에 하나뿐인 내 글씨"는 대체 불가·감정가치 큼. 청첩장/브랜드/굿즈 수요 | 高(AI API) |
| **한글 포함** | 라틴→한글 2,350자(KS) ~ 11,172자 확장 | 한글 손글씨 폰트는 수작업 단가 수십만 원. 자동화 가치가 직접적 | 高 |
| **상업용 라이선스** | 상업 사용 허가증(PDF) + 약관 | 사업자·디자이너는 라이선스 없으면 못 씀. 면책이 곧 가치 | 低(서류/DB) |
| **TTF/OTF·웹폰트킷 내보내기** | TTF/OTF/WOFF2 + @font-face CSS 묶음 | 실무는 설치/웹배포 포맷 필수. WOFF 미리보기만으론 부족 | 低(이미 보유) |
| **워터마크 제거** | 무료 산출물의 표식 제거 | "깔끔한 완성본" 욕구. 가장 마찰 적은 결제 트리거 | 低 |
| **고해상도/대용량 글리프셋** | 특수문자·구두점·확장 라틴·자모 풀셋 | 실사용 시 빠진 글자 = 치명적. 완결성에 지불 | 中 |
| **무드 프리셋 묶음** | "빈티지/귀여움/손글씨" 등 큐레이션 변형 세트 | 비전문가의 선택 피로 해소. 즉시 결과 | 低(파라미터 조합) |
| **빠른 생성(우선순위 큐)** | AI 생성 대기열 우선 처리 | 무료티어 1인 운영의 대기시간을 돈으로 단축 | 中(큐 설계) |
| **폰트 보관/버전관리** | 클라우드 저장·재다운로드·이력 | 재작업 방지, 자산화. 구독 락인 효과 | 中(스토리지/DB) |
| **팀/브랜드킷** | 폰트+컬러+로고 묶음 공유 | B2B·소규모 브랜드 객단가 상승 | 高(후순위) |

> 핵심 통찰: **변형(보간) 결과물 자체엔 큰 돈을 받기 어렵다**(기존 폰트의 파생). 결제는 ① 창작성(내 손글씨/AI), ② 실무 사용권(라이선스+포맷+워터마크 제거), ③ 시간(우선순위/보관)에서 나온다.

## 2. 무료 vs 유료 라인 (게이팅 지점)

| 항목 | 무료 (입소문·SEO 유입) | 유료 (전환 트리거) |
|------|----------------------|-------------------|
| 생성 방식 | 전통 파라미터 변형 | AI 손글씨 완성 |
| 문자셋 | 라틴(a–z/대문자/숫자) | 한글 + 확장 라틴/특수문자 |
| 미리보기 | 실시간 무제한 | 동일(유입 미끼 유지) |
| 다운로드 | WOFF, **워터마크 포함**, 개인용 한정 | TTF/OTF/WOFF2 풀킷, 워터마크 없음 |
| 라이선스 | 개인·비상업 | 상업용 라이선스 증서 |
| 무드 프리셋 | 기본 2~3종 | 프리셋 묶음 전체 |
| 생성 속도 | 일반 큐 | 우선순위 큐 |
| 보관 | 없음(즉시 다운로드) | 클라우드 보관·버전관리 |

**게이팅 트리거(전환 순간) 설계**
1. 미리보기에서 **한글 입력 시** → "한글 완성은 Pro" 잠금.
2. 다운로드 버튼 → 무료는 워터마크/개인용 고지, **"상업용·워터마크 제거"** 업셀 모달.
3. TTF/OTF·웹킷 선택 → 유료 포맷 표시.
4. 변형으로 "원하는 느낌이 안 나옴" → "AI로 내 손글씨처럼 완성" CTA.

> 원칙: **유입 미끼(미리보기·라틴 변형·소량 무료)는 절대 막지 않는다.** SEO·공유로 들어온 사용자가 "완성/상업/한글" 벽에서만 결제하도록.

## 3. 요금제 설계 (사용량 vs 구독 vs 프리미엄 혼합)

**구조 원칙**: AI 원가가 큰 1회성 생성은 **사용량 과금(크레딧)**, 라이선스·보관 등 마진 높은 권리는 **구독/단건 프리미엄**. → 적자 방지 + 예측 가능.

### 티어 예시 (원화)

| 티어 | 가격(안) | 포함 | 과금 모델 |
|------|---------|------|----------|
| **Free** | 0원 | 라틴 변형, 미리보기 무제한, WOFF(워터마크·개인용), 프리셋 2종 | — |
| **단건 다운로드** | 3,900~5,900원/폰트 | 워터마크 제거 + TTF/OTF·웹킷 + 상업 라이선스 1건(라틴/변형) | 사용량(단건) |
| **AI 생성 크레딧** | 9,900원 = 라틴 완성 약 3건 / 한글 완성 1건 | AI 손글씨 완성. **한글 1건 = 라틴 약 3건 분 원가** 반영 | 사용량(크레딧) |
| **Pro 구독** | 9,900원/월 (연 99,000원) | AI 크레딧 매월 N개 + 모든 포맷·프리셋 + 워터마크 제거 + 클라우드 보관/버전 | 구독 |
| **상업/브랜드** | 29,000원/월~ | Pro + 확장 상업 라이선스(직원/클라이언트 사용) + 브랜드킷 + 우선순위 큐 | 구독(B2B) |

### AI 원가 전가 구조
- AI 1회 생성 호출당 **예상 API 원가 + 마진**을 **크레딧 1~N개**로 환산.
- **한글은 글자 수 × 토큰/이미지 비용**이 라틴보다 크므로 **크레딧 가중치 차등**(예: 라틴 1, 한글 3~5).
- 생성 **호출 전 비용 확인 모달**(크레딧 차감 안내) → 환경변수 AI 토글 OFF 시 기능 숨김.
- 크레딧 **선구매(prepaid)** → 외부 API 후불 폭증 리스크를 사용자 선결제로 상쇄.

## 4. 경쟁/시장 스캔 (아는 범위, 불확실은 "가정" 표기)

| 영역 | 현황(가정 포함) | 우리 차별점 |
|------|----------------|------------|
| 손글씨→폰트 자동화 | Calligraphr(템플릿에 손글씨 써서 폰트화, 글자별 직접 작성), Fontself(일러스트 플러그인) — **사용자가 글자 다 그려야 함** | **몇 글자만 → AI가 나머지 완성**(작성 부담 ↓) |
| AI 폰트 생성 | (가정) 일부 스타트업 존재하나 한글·한국 결제·토스 미니앱 대응은 약함 | 한국어 UX + 한글 + 인앱결제 친화 |
| 한글 폰트 시장 | 글자 수 많아 제작 단가 높음(상용 한글폰트 수만~수십만원, 손글씨 외주 수십만원). 산돌/윤디자인 등 B2B 구독 존재 | **개인 손글씨 한글 폰트를 저가 자동화** — 빈 시장 |
| 무료 변형/생성기 | Google Fonts 변형, 캔바 등 | 우리는 "다운로드+상업 라이선스+내 손글씨"로 차별 |

**한글 시장 특수성**: ① 글자 수 폭발(완성형 2,350 / 조합형 11,172) → 자동화 가치가 곧 차별점. ② 청첩장·다이어리·굿즈·자영업 간판 등 "내 글씨" 감성 수요 강함. ③ 라이선스 분쟁 민감 → **상업 라이선스 증서**가 신뢰 자산.

## 5. 토스 미니앱 정합 (제약 하 가능한 수익화만)

| 토스 제약 | 영향 | 대응(설계 포인트) |
|-----------|------|------------------|
| **인앱결제만**(외부 PG 불가) | 단건·크레딧·구독 모두 토스 인앱결제로 | 결제 모듈을 **인터페이스로 추상화**(`PaymentProvider`): 웹=토스페이먼츠/스트라이프, 미니앱=토스 인앱결제. 가격·상품 카탈로그는 공용. |
| **외부 광고망 불가** | 광고 수익 모델 제한 | 광고 의존 버리고 **기능 과금 중심**. 미니앱은 앱인토스 광고만(보조) |
| **완결형 구조**(외부 다운로드 불가) | 미니앱 내에서 폰트 받기/사용 완결 | 미니앱은 **앱 내 보관함 + 공유**로 완결, 외부 링크 다운로드 금지. 웹앱은 직다운 허용(별도 코드경로) |
| **AI 생성 표시 의무** | AI 결과물 라벨/워터마크 필수 | AI 산출물엔 처음부터 "AI 생성" 메타·배지·(무료는)워터마크. `generatedBy: "ai"` 플래그를 core 계약에 추가 권장 |
| **로그인=토스** | 자체 로그인과 분리 | **AuthProvider 추상화**: 웹=이메일/소셜, 미니앱=토스 로그인. 사용자 ID 매핑 테이블 |
| 유사앱 중복 금지 | 폰트/이모티콘은 핵심기능 다르면 별도 앱 가능 | 앱 정체성 "폰트 생성"으로 명확히 |

**추상화 체크리스트(코드 짤 때부터)**: 결제·로그인·다운로드/공유 3가지를 **provider 인터페이스**로 분리 → 웹/미니앱 동시 운용. (실제 구현은 별도 작업, 본 문서는 설계만.)

## 6. 측정 / 그로스

**핵심 퍼널**: 방문(SEO/공유) → 생성(미리보기) → 다운로드 시도 → **게이트 도달** → 결제 → 재방문/보관.

| 단계 | 핵심지표(KPI) |
|------|--------------|
| 유입 | 검색 유입수, 공유 OG 클릭률 |
| 활성 | 생성(미리보기) 시도율, 슬라이더 조작수 |
| 전환 트리거 | 게이트 도달률(한글/상업/포맷), 업셀 모달 노출→클릭 |
| 결제 | 다운로드→결제 전환율, 단건 vs 구독 비중, 크레딧 소진율 |
| 유지 | 재다운로드율, 구독 갱신율, 보관함 사용률 |

**그로스 루프**
- **공유 OG**: 생성된 폰트로 렌더된 미리보기 이미지를 OG 카드로 → SNS 공유 시 "이거 어떻게 만들었어?" 유입. (SSR로 동적 OG 생성, `sitemap.ts`/`robots.ts` 이미 존재 → OG 라우트 추가 권장.)
- **SEO**: "손글씨 폰트 만들기 / 내 글씨 폰트 / 한글 손글씨 폰트" 키워드 랜딩.
- **무료 산출물 워터마크**가 곧 광고(워터마크에 서비스명) → 바이럴 + 워터마크 제거 업셀.

## 7. 가치–노력–수익 우선순위 매트릭스

| 우선순위 | 기능 | 가치 | 노력 | 수익기여 | 근거 |
|---------|------|------|------|---------|------|
| **P0(즉시)** | 상업 라이선스 + 워터마크 제거 + TTF/OTF·웹킷 내보내기 | 高 | **低**(엔진 보유) | 中~高 | AI 없이 **지금 매출 시작** 가능. 단건 과금 |
| **P0** | 결제·로그인·다운로드 **provider 추상화** | (기반) | 中 | (전 매출 전제) | 웹/토스 동시대비, 나중 재작업 방지 |
| **P1** | 무드 프리셋 묶음 | 中 | 低 | 中 | 파라미터 조합만으로 객단가↑, 비전문가 흡인 |
| **P1** | 공유 OG 미리보기 + SEO 랜딩 | 高(유입) | 中 | (간접) | 결제 모수 자체를 키움 |
| **P2(핵심)** | **AI 손글씨 완성(라틴)** + 크레딧 과금 | 高 | 高 | 高 | 진짜 차별화. 원가 전가 구조 검증 |
| **P2** | **한글 포함 AI 완성** | 매우高 | 매우高 | 매우高 | **1순위 가치**, 단 기술난이도로 AI 검증 후 |
| **P3** | 폰트 보관/버전관리(구독 락인) | 中 | 中 | 中(LTV) | 구독 갱신·재방문 |
| **P3** | 팀/브랜드킷(B2B) | 中 | 高 | 中~高 | 객단가 높지만 수요 검증 후 |

**실행 순서 요약**: P0(라이선스·포맷·워터마크 제거로 무코스트 매출) → 추상화 기반 → P1(프리셋·공유로 모수 확대) → P2(AI 라틴→한글, 크레딧 원가전가) → P3(보관·B2B로 LTV).
