# 손글씨 코어 — "내가 그린 글씨 → 진짜 글씨체" (방향 재설정)

> 마스터 지적: 기성 가변폰트를 슬라이더로 변형하는 건 독자성·유용성이 약하다.
> **진짜 코어 = 직접 그린 획을 그대로 폰트 글리프로 굽고, 개성은 살리되 다듬어 글씨체로 만든다.**
> 슬라이더 변형은 "빠른 시작 샘플"로 강등.

## 증명됨 (PoC 통과)
그린 획(중심선 폴리라인) + 펜 두께 → 좌우 오프셋 외곽선 → `fontTools.fontBuilder`/`TTGlyphPen`로 glyf 조립 → 진짜 TTF/WOFF 재오픈 성공. **potrace/OpenCV 불필요**(stroke 기반). 무료티어 부담 작음.

## 코어 파이프라인
1. **그리기(캡처)**: 글자별 가이드 그리드(베이스라인/x-height/cap). 포인터 경로를 **벡터 획(폴리라인)** 으로 캡처(이미 벡터 → 래스터 벡터화 불필요). 한 글자 = 여러 획. 자유도 높게(획수 제한 없음).
2. **다듬기(refine, 디자이너 고찰을 간편·고자유도로)**:
   - `smoothing`(베지어 정리: 손떨림 제거하되 형태 유지)
   - `nib`(획 두께/펜촉), 선택적 taper(필압 흉내)
   - `straighten`(기울기/베이스라인 보정)
   - `spacing`(자간/사이드베어링), 메트릭 정규화
   - 모두 0이면 **날것 그대로**(개성 100%), 올리면 정제. "못생긴 개성 → 살린 글씨체".
3. **조립(엔진)**: 획→외곽선→glyf, 안 그린 글자는 비움(또는 베이스폰트 폴백 옵션). WOFF/TTF/OTF 출력. `generatedBy: "handwriting"`(진짜 내 글씨 — 정직).
4. **프리뷰/받기**: 내가 그린 글자들로 문장 렌더 + 다운로드.

## MVP 범위
- 라틴 소문자 a–z 우선(원하는 만큼만 그려도 됨). 한글은 자모 입력(다음).
- 빠른 시작: 몇 글자만 그려도 즉시 폰트 미리보기.

## 싸인 재설정 (독자 변환)
- 이름 렌더 ❌ → 그린/입력을 **스탬프(낙관)화**(원형/사각 도장, 단색 실루엣), **획화(stroke-ification)**, 캐릭터 획 변환 등 독자 메커니즘. 같은 stroke 엔진 재사용.

## 계약 (packages/core)
- `GlyphStroke{points}` / `DrawnGlyph{char,strokes}` / `RefineParams` / `HandwritingRequest`.
- 응답 `generatedBy`에 `"handwriting"` 추가.
- 스트로크 페이로드 크기 상한(무료티어 메모리).
