# 글씨의 캐릭터화 + 나만의 이모티콘/이모지 (Phase 2 아이디어)

> 작성: 캐릭터·이모지 아이디어 에이전트 / 코드 수정 없음. 이 문서는 빌더 착수용 설계 메모다.
> 현재 코드 기준: 엔진은 Recursive 가변폰트를 `wght/slnt/CASL`로 인스턴스화 → 라틴 a–z 서브셋 → WOFF/TTF. 프론트엔드엔 `DrawingCanvas`(PNG dataURL 추출 가능, 현재 폰트에 미반영)와 슬라이더/프리뷰가 이미 있다.
> 계약은 `packages/core/src/index.ts`의 `FontParams = {weight, slant, curvature}` 단일 출처. 의존성: `fonttools==4.63.0`(현재 Pillow/OpenCV/Potrace **미설치**).
> 기획서(`docs/font-project-plan.md`) Phase 3에 "이모티콘·문자 이모티콘 자동 생성"이 명시돼 있다.

## 한 줄 결론 + 1순위 권장

**"폰트"가 아니라 "스티커 PNG 세트"가 가장 빠르고, 가장 실용적이며, 가장 차별적이다.** 컬러 이모지 폰트(COLR/SVG)는 만들기는 되지만 사용자가 "쓸 데"가 거의 없다(OS·앱이 커스텀 이모지 폰트를 설치/렌더하지 못함). 반면 **투명 PNG 스티커 팩은 카톡·인스타·디스코드에 바로 올라간다.**

**1순위 권장: (c) "한 번 그리면 표정/색 변주 N종 자동 생성 → 투명 PNG/APNG 스티커 팩 내보내기"** — 사용자가 캐릭터 하나(또는 글자 하나)를 그리면 절차적 변형(표정 파츠 교체·색 변주·테두리·간단 모션)으로 8~24종 세트를 자동으로 뽑아 ZIP/스티커팩으로 내려준다. 비AI·비용 0, 기존 `DrawingCanvas` 재사용, 결과물이 즉시 공유 가능. 이게 폰트 본체와 **목적·핵심기능이 명확히 다르므로**(토스 유사앱 중복 규정 통과) **별도 앱 `apps/sticker`로 분리 권장.**

## 1. 개념 정의/분류

| # | 이름 | 결과물 형태 | 어디에 쓰나 | "내 창작물" 정도 | 비AI 가능 |
|---|------|------------|------------|----------------|----------|
| (a) | 글자→캐릭터화 | 글리프에 눈/표정/팔다리 파츠를 합성한 **캐릭터 폰트**(WOFF/TTF) 또는 PNG | 로고·타이틀·굿즈 텍스트 | 글자형은 기성/내 글씨, 장식은 규칙 합성 | O |
| (b) | 컬러 이모지 폰트 | **COLR/CPAL** 또는 **SVG-in-OpenType** 폰트 | 거의 없음(아래 ⚠). 자사 웹 프리뷰 한정 | 부분 | △(만들 순 있음) |
| (c) | 이모티콘/스티커 팩 | **투명 PNG / APNG / WebP 세트**(+팩 메타) | 카톡·인스타·디스코드·텔레그램 공유 | 높음(내 그림 + 변주) | O |
| (d) | "내 글씨 + 그림" 조합 스티커 | 내 손글씨 텍스트 + 내 그림을 합성한 PNG/APNG | 짤·밈·인사말 스티커 | 가장 높음(글씨도 그림도 내 것) | O |

### (a) 글자 → 캐릭터화
글리프 외곽선 위/주변에 **눈·입·볼터치·팔다리** 같은 파츠를 규칙으로 얹어 글자를 캐릭터처럼 보이게 한다.
- 결과물 1: **캐릭터 폰트** — 각 글자가 캐릭터인 데코 폰트. 단색이면 기존 WOFF/TTF 파이프라인 재사용 가능(외곽선에 파츠 외곽선을 union).
- 결과물 2: **글자 캐릭터 PNG** — 색·표정까지 넣으면 폰트보다 PNG가 자연스럽다(컬러 폰트의 활용처 부족 문제 회피).
- 용도: 닉네임 로고, 타이틀, 굿즈("내 이름 캐릭터").

### (b) 컬러 이모지 폰트 (COLR/CPAL · SVG table) — ⚠ 활용처 함정
- 기술적으로 fontTools로 **만들 수는 있다**(아래 2절). 하지만 **사용자가 설치해서 쓸 수 없다**: OS 이모지(애플/구글)는 시스템 폰트로 고정이고, 일반 앱은 사용자가 만든 컬러 이모지 "폰트"를 이모지 자리에 넣어주지 않는다.
- 현실적 활용처는 **우리 웹앱 내부 프리뷰**와 **웹사이트에 `@font-face`로 임베드**(개발자 한정) 정도. 일반 사용자 가치 낮음.
- **권고: (b)는 "기술 데모/내부 렌더링용"으로만, 사용자 출력 1순위에서 제외.** 사용자에겐 (c) PNG로 주는 게 정답.

### (c) 이모티콘/스티커 팩 (PNG/APNG/WebP 세트)
- 한 캐릭터를 **표정·포즈·색**으로 변주해 **세트**로 만들고, 투명 배경 PNG(정지) + APNG/WebP(움직임)로 내보낸다.
- 카톡·인스타 스토리 스티커·디스코드 커스텀 이모지·텔레그램 스티커가 전부 PNG/WebP/APNG 기반 → **곧바로 쓸 수 있는 유일한 형식.**
- **이게 1순위.**

### (d) "내 글씨 + 그림" 조합 스티커
- 기존 폰트앱의 **내 손글씨 폰트(라틴/추후 한글)** + (c)의 **내 그림 캐릭터**를 한 캔버스에 합성("오늘도 화이팅" 같은 손글씨 + 캐릭터).
- 두 앱(font·sticker)의 결과를 한 곳에서 합치는 **크로스-앱 시너지**. font 앱의 차별점을 sticker가 흡수.

### 컬러 폰트: fontTools 실현성
- **COLR/CPAL v0**: `fontTools.colorLib.builder.buildCOLR` + `buildCPAL`로 "글리프 = 여러 단색 레이어의 스택"을 만든다. 무료티어 부담 거의 없음(외곽선 데이터만 추가). 단순 컬러 이모지에 충분.
- **COLR v1**: 그라데이션·변형 지원(`PaintLinearGradient` 등). v0 대비 복잡, 렌더 지원도 v1이 더 좁음.
- **SVG-in-OpenType (`SVG ` 테이블)**: 각 글리프에 SVG 문서를 통째로 임베드. 표현력 최고지만 파일이 커지고, 폴백용 단색 glyf를 병행해야 하며 렌더러 호환이 들쭉날쭉.
- **결론**: 만들기는 다 된다. 문제는 §1(b)에서 말한 **활용처 부족**. 그래서 컬러 폰트는 후순위로 두고, 같은 레이어 데이터를 **PNG 래스터로 합성해 (c)로 출력**하는 게 ROI가 높다.

### 스티커 PNG 세트: 절차적(비AI) N종 자동 생성 — 핵심 엔진
입력 1개 → 출력 N개를 만드는 변주 축(전부 비AI):
1. **표정 파츠 교체**: 눈/입 파츠 라이브러리(예: 웃음·윙크·하트눈·우는·놀람·화남 ×입모양)에서 조합 → "기쁨/슬픔/사랑/분노/놀람…" 세트. 사용자가 그린 캐릭터에서 얼굴 영역(고정 앵커 또는 사용자가 클릭 지정)에 파츠를 합성.
2. **색 변주**: HSV 회전/팔레트 스왑으로 같은 캐릭터를 N색.
3. **테두리/스티커화**: 알파 외곽에 흰색/검정 아웃라인(거리변환 또는 Canvas shadow 누적), 둥근 배경 칩.
4. **간단 모션(APNG/WebP)**: 통통 튀기(scale bounce), 좌우 흔들기(rotate), 깜빡임(파츠 토글) 같은 **프로그램 모션** 2~6프레임. 손그림 프레임 애니메이션이 아니라 변환 기반이라 비용 0.
5. **텍스트 합성(d)**: 말풍선 + 내 손글씨/기성 텍스트("ㅇㅋ" "화이팅").

**서버 0안(권장 시작점)**: 위 합성을 전부 **브라우저 Canvas/OffscreenCanvas**에서 수행 → 서버는 ZIP 패키징/팩 메타 검증만(또는 그것도 클라). 무료티어 부담 0, OpenCV/Pillow 불필요. APNG는 클라 라이브러리(예: upng/apng-asm 류)로 인코딩.
**서버안(필요 시)**: Pillow로 합성/리사이즈/APNG. CPU만 쓰면 비용 0이나 메모리 피크 주의(현재 세마포어=3 패턴 재사용).

### 절차적 캐릭터화: 파츠 라이브러리 + 배치 규칙
- **파츠**: SVG(벡터, 스케일 자유) 권장. `parts/eyes/*.svg`, `parts/mouth/*.svg`, `parts/deco/*.svg`(볼터치·반짝이·팔다리).
- **배치 규칙**: 캐릭터 바운딩박스 정규화 → 얼굴 앵커(상단 40~60% 영역) → 눈은 좌우 대칭 슬롯, 입은 눈 아래 슬롯, 데코는 모서리 슬롯. 글자→캐릭터화(a)는 글리프 외곽선의 바운딩박스를 앵커로 동일 규칙 적용.
- **표정 = 파츠 조합 프리셋**: `{eyes, mouth}` 튜플 8~12개를 "감정 프리셋"으로 정의 → 한 번에 세트 생성.

### 사용자 그림 윤곽선 추출/벡터화 (선택)
- 캐릭터화를 "내 그림 위에" 하려면 PNG 그대로 합성하면 충분(벡터화 불필요). **벡터화는 (a) 캐릭터 폰트를 만들 때만 필요** → OpenCV/Potrace 도입은 그 단계로 미룬다(CLAUDE.md "무거운 ML/이미지 의존성 도입 전 메모리 검토" 준수).

### AI 경로 (유료·기본 OFF)
- 그림→스타일 변형 다수(다른 화풍·고품질 표정 세트). 외부 이미지 생성 API.
- CLAUDE.md 철칙: **환경변수 ON/OFF + 호출 전 비용 경고 주석 + 기본 OFF + 유료 사용자 전용 + 결과물 "AI 생성" 라벨 강제**(`generatedBy:"ai"`, 워터마크/배지). 토스 대비 필수.

## 3. 모노레포 신규 앱 여부 — 권고: **별도 앱 `apps/sticker`**

CLAUDE.md "새 앱 추가 규칙"과 토스 "테마만 다른 유사앱 중복 금지(목적·핵심기능이 명확히 다르면 별도 앱 가능)"를 함께 고려.

- **폰트앱 내부 기능으로?** → 폰트앱의 핵심은 "글자 스타일 → 폰트 파일". 스티커는 "그림 → 이미지 세트 공유"로 **입력·출력·사용맥락이 다르다.** 한 앱에 욱여넣으면 UX가 흐려지고, 토스 전환 시 오히려 "한 앱이 두 목적"이라 심사 모호.
- **권고: `apps/sticker/{frontend,engine}` 신규 앱.** 근거:
  - 핵심기능이 폰트(파일 생성)와 명확히 다름 → 토스 유사앱 규정 통과 유리.
  - 공통 로직(드로잉 캔버스, 색/표정 합성, 내보내기)은 **복붙 금지 → `packages/`로**:
    - `packages/ui`: `DrawingCanvas`를 font 앱에서 끌어올려 공용화(현재 `apps/font/frontend/components`에 있음 → Shared-Agent 경유 이동).
    - `packages/core`: 스티커 계약(아래 6절) 추가.
    - 신규 `packages/sticker-kit`(또는 core 하위 모듈): 파츠 라이브러리 메타·표정 프리셋·합성 규칙·팩 규격 상수.
  - `home/`에 sticker 카드/링크 추가, `infra/nginx`에 `메인도메인/sticker` 라우팅 추가(CLAUDE.md 규칙대로).
- **단, (d) "내 글씨+그림"은 두 앱을 잇는 지점** → font가 만든 폰트를 sticker가 `@font-face`로 불러 텍스트 합성. 데이터 계약만 공유하면 됨.
- **착수 비용 절감 팁**: 1차는 `apps/sticker/frontend`만(서버 0안, 클라 합성) 만들고, `engine`은 ZIP/APNG가 서버로 가야 할 때 추가.

## 4. 효율적 창작 UX — "적은 입력, 많은 결과"

- **한 번 그리면 N종 자동(핵심)**: 캐릭터 1개 + 얼굴 앵커 1회 지정 → 감정 프리셋 8~24종을 한 화면에 그리드로 즉시 미리보기. "이 12개를 팩으로 받기" 버튼.
- **프리셋/템플릿**:
  - 감정 팩(기쁨/슬픔/사랑/분노/놀람/졸림/오케이/화이팅…), 일상 팩(인사·맞장구·거절), 시즌 팩(생일·크리스마스).
  - 캐릭터 템플릿(동그라미 몸체·네모 몸체)을 제공해 "그림 못 그려도" 표정만 골라 세트 완성.
- **실시간 변주 슬라이더**: 색상(HSV), 아웃라인 두께, 둥글기, 모션 속도. 기존 폰트앱 슬라이더 UX 패턴 재사용.
- **부분 재그리기**: 특정 표정만 마음에 안 들면 그 칸만 다시 그려 덮어쓰기(한글 자모조합 보정 입력과 동일 사상).
- **내보내기 한 곳에서**: 투명 PNG(개별/ZIP) · APNG/WebP(모션) · 스티커팩(메타 포함) · (선택)캐릭터 폰트 WOFF.
- **접근성**: 현재 `DrawingCanvas`가 키보드 미지원이므로(주석 명시) 템플릿+파츠 선택만으로도 세트를 완성할 수 있는 비드로잉 경로 제공.

## 5. 출력/공유 규격 (아는 범위, 모르는 건 "가정" 표기)

> 플랫폼 규격은 수시로 바뀐다. 아래 수치는 빌더 착수용 기준이며 **배포 직전 각 플랫폼 공식 가이드 재확인 필수(가정 포함).**

- **카카오톡 이모티콘(가정/대략)**: 멈춰있는 이모티콘 통상 **32종**, 움직이는 이모티콘 **24종** 단위가 흔함. 캔버스 360×360px 안팎, 투명 PNG, 움직임은 WebP/APNG. **이건 카카오 "이모티콘 스튜디오" 정식 입점 규격**이므로 우리 앱은 "카톡에 보낼 수 있는 PNG/움짤"을 만들어 주되, **공식 상품 등록 규격을 강제하기보단 그 규격에 맞춘 export 옵션을 제공**하는 선이 현실적. (정확 수치·해상도는 입점 가이드 재확인.)
- **인스타 스토리 스티커**: 투명 PNG를 카메라롤 저장 → 스토리에서 불러 쓰기. 권장 짧은 변 ~512px+ 투명 PNG(가정).
- **디스코드 커스텀 이모지**: PNG/GIF/WebP, **256KB 이하**, 표시 128×128(업로드는 더 커도 됨). 작은 용량 최적화 export 필요(가정 수치, 재확인).
- **텔레그램 스티커(가정)**: 512×512 투명 PNG/WebP, 한 변 512 고정. 팩 단위.
- **공통 권장 export 프리셋**: `512×512 투명 PNG`(범용), `360×360 PNG/APNG`(카톡향), `128×128 작은 용량`(디스코드), `ZIP 묶음`, `APNG/WebP 모션`.
- **OG 공유 루프(SEO·바이럴)**: 생성한 세트의 대표 이미지를 OG 이미지로 한 공유 페이지(`/sticker/s/[id]`) 자동 생성 → 카톡/SNS 링크 공유 시 썸네일 노출 → 유입. `packages/seo` 규칙(고유 title/description/OG, SSR, JSON-LD) 준수.

## 6. 무료 vs 유료 라인 + 계약/데이터 모델 영향

### 무료 / 유료
- **무료(비AI, 비용 0)**: 절차적 표정/색/테두리 변주, 모션 프리셋, 투명 PNG/ZIP, 워터마크(선택), 저해상도/세트 수 제한.
- **유료**: 고해상도·무손실, 워터마크 제거, 대용량 세트, **AI 스타일 변형**(외부 API, 환경변수 OFF 기본·"AI 생성" 라벨), 상업 라이선스, 캐릭터 폰트(WOFF/TTF) 다운로드.
- 토스 전환 대비: 결제=인앱결제 추상화, 외부 다운로드 강제 금지(앱 내 완결), 외부 광고망 직접삽입 금지(기존 정책 그대로).

### `packages/core` 추가 제안 (하위호환 — 기존 `FontParams`/`GenerateRequest` 불변, 스티커용 신규 타입 분리)

```ts
// ── 스티커/이모지 계약 (apps/sticker, packages/sticker-kit 공유) ──

/** 출력 형식. png=정지, apng/webp=모션, zip=세트 묶음, woff=캐릭터 폰트(유료) */
export type StickerFormat = "png" | "apng" | "webp" | "zip" | "woff";

export const STICKER_FORMATS: Record<StickerFormat, { mime: string; ext: string; animated: boolean }> = {
  png:  { mime: "image/png",  ext: "png",  animated: false },
  apng: { mime: "image/apng", ext: "png",  animated: true  },
  webp: { mime: "image/webp", ext: "webp", animated: true  },
  zip:  { mime: "application/zip", ext: "zip", animated: false },
  woff: { mime: "font/woff",  ext: "woff", animated: false },
};

/** 공유 대상별 export 프리셋(크기/용량). 수치는 배포 전 플랫폼 가이드 재확인. */
export type SharePreset = "universal512" | "kakao360" | "discord128" | "telegram512";

/** 표정 프리셋 = 눈/입 파츠 조합 + 라벨 */
export interface EmotionPreset {
  id: string;            // "joy" | "sad" | "love" | ...
  label: string;         // "기쁨" 등 (UI 표시)
  eyes: string;          // 파츠 라이브러리 키
  mouth: string;
  deco?: string[];       // 볼터치·반짝이 등
  motion?: MotionPreset; // 모션 세트일 때
}

/** 비AI 프로그램 모션(프레임 합성 없이 변환으로 생성) */
export interface MotionPreset {
  kind: "bounce" | "shake" | "blink" | "pulse" | "none";
  frames: number;        // 2~6
  fps: number;
}

/** 한 번 그리고 → N종 변주 요청 */
export interface StickerSetRequest {
  baseImagePng: string;          // 사용자가 그린 캐릭터(dataURL). MAX_IMAGE_PNG_BYTES 재사용
  faceAnchor?: { x: number; y: number; w: number; h: number }; // 얼굴 영역(정규화 0~1)
  emotions: string[];            // 적용할 EmotionPreset id 목록
  colorVariants?: number;        // HSV 회전 변주 수(0=원색만)
  outline?: { width: number; color: string } | null;
  format: StickerFormat;
  preset?: SharePreset;          // 크기/용량 프리셋
  useAi?: boolean;               // 기본 false. true는 유료+환경변수 ON일 때만
}

export interface StickerSetResponse {
  items: { emotionId: string; dataBase64: string; format: StickerFormat }[];
  packBase64?: string;           // zip일 때
  generatedBy: "procedural" | "ai";   // 정직성 라벨(토스 대비)
  watermarked: boolean;
}
```

- 신규 상수: `EMOTION_PRESETS`(기쁨/슬픔/사랑/분노/놀람/졸림/OK/화이팅…), `MAX_STICKER_SET_SIZE`(무료 N 제한), `SHARE_PRESET_SPECS`(프리셋별 px/용량 상한). `MAX_IMAGE_PNG_BYTES`는 기존 값 재사용.
- 파츠 라이브러리 메타(눈/입/데코 키 목록, SVG 경로)는 `packages/sticker-kit`에 두고 프론트·엔진이 동일 키를 참조.
- `generatedBy`는 **항상 응답에 포함**해 클라가 "AI 생성"/"자동 합성" 라벨을 강제 노출(한글 doc의 `generatedBy` 정직성 패턴과 동일 사상).

## 요약 (빌더용)

- **1순위(S1)**: "한 번 그리면 표정/색 변주 N종 자동 → 투명 PNG/ZIP 스티커 팩". **브라우저 Canvas 합성(서버 0)** 으로 시작 → 비용 0, 기존 `DrawingCanvas` 재사용, 카톡·인스타·디스코드에 바로 공유 가능. 가장 실용적·차별적.
- **컬러 이모지 폰트(b)는 만들 수는 있으나(fontTools COLR/CPAL·SVG table) 사용자가 쓸 데가 거의 없다 → 후순위, 내부 데모용.** 같은 데이터를 PNG로 출력하는 게 ROI 우위.
- **신규 앱 권고**: `apps/sticker/{frontend,engine}` 분리(토스 유사앱 규정 통과 유리, 핵심기능이 폰트와 다름). 공통 로직은 `packages/ui`(DrawingCanvas 공용화)·`packages/core`(스티커 계약)·신규 `packages/sticker-kit`(파츠/프리셋/규격)로. `home`·`infra/nginx`에 sticker 추가.
- **UX**: 적은 입력 많은 결과(감정 프리셋 8~24종 그리드 즉시 프리뷰), 캐릭터 템플릿(비드로잉 경로), 색/아웃라인/모션 슬라이더, 칸별 재그리기.
- **무료/유료**: 무료=절차적 변주·PNG/ZIP(비용 0), 유료=고해상도·워터마크 제거·캐릭터 폰트·AI 변형. AI는 환경변수 OFF 기본·유료 전용·"AI 생성" 라벨 강제.
- **계약**: `StickerFormat`, `StickerSetRequest/Response`, `EmotionPreset`, `MotionPreset`, `SharePreset` 신규(기존 FontParams 불변, 하위호환). `generatedBy`("procedural"|"ai") 항상 포함해 정직성 라벨.
- **공유 규격**: 카톡 멈춤32/움짤24·360px 안팎, 디스코드 256KB·128px, 텔레그램 512px(전부 **가정, 배포 전 재확인**). export 프리셋으로 대응. OG 공유 페이지로 유입 루프.
- **의존성 주의**: 시작은 클라 합성으로 서버 의존성 0. 캐릭터 폰트·서버 합성·벡터화 단계에서만 Pillow/OpenCV/Potrace 도입(CLAUDE.md대로 메모리 영향 사전 검토).
