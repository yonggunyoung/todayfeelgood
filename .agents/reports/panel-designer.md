**감빛 테라코타 전환은 "AI 기본색" 리스크를 실제로 끊어냈고 구조·타이포·형태 블랙리스트는 코드상 0건 — 하지만 지금 빌드는 "잘 만든 iOS 클린 템플릿"에서 멈춰 있어 마스터가 요구한 "여지껏 없던 조합"의 독창성은 아직 약하고(문방구·획·너굴이 컨셉이 표면에 안 박혀 있음), 스튜디오 7단 카드 과밀이 초심자 진입장벽으로 남아 있다.**

## Blocker (다음 라운드 전 반드시)

### B1. 스튜디오 컨트롤 7단 카드 수직 과밀 — 초심자 진입장벽 (오케스트레이터 시드 확정)
- **위치:** `app/studio/FontStudio.tsx:205-308`, `FontStudio.module.css:22-51`.
- **문제:** 좌측 `minmax(0,24rem)` 단일 열에 `.group` 카드가 **7개 연속**(문자체계 / 무드 프리셋 / 변주 갤러리 / 세부 조절 / 스케치(준비 중) / 이미지 효과(PNG) / 받아 가기). 그 안에 `ParameterPanel`만 슬라이더 **9개**(기본3+형태5+손맛3, `ParameterPanel.tsx:30-49,101-137`). 처음 연 사용자는 "어디부터 만지지?"에 압도된다. 모든 카드가 같은 `shadow-sm`+`r-lg`+동일 padding이라 **위계가 평평**(무엇이 1순위 액션인지 안 보임). "준비 중" 스케치와 "PNG 전용" 효과가 핵심 조작(슬라이더)과 같은 비중으로 끼어 흐름을 끊는다.
- **개선 지시:**
  1. **3단계 위계로 재편**: (a)빠른 시작 = 무드 프리셋 + 변주 갤러리(누르면 끝), (b)세부 조절 = 슬라이더(기본 펼침), (c)고급/실험 = 스케치·PNG효과를 **접힌 `<details>` 아코디언**으로. 슬라이더 그룹(곡선·형태 5개, 손맛 3개)도 접기 기본값 = "곡선·형태" 접힘.
  2. **"준비 중" 스케치 카드는 최하단 또는 접힘으로** 강등 — 첫 화면에서 미완성 기능이 보이면 신뢰가 깎인다.
  3 데스크톱 좌측 패널을 카드 사이 `gap`만 주지 말고 **섹션 헤더 위계 차등**(빠른 시작은 강조 배경, 고급은 ghost). 모든 group이 동일 surface인 현 상태는 정보 위계 0.
  4 디렉션 4-3이 명시한 **모바일 하단 시트/탭바**가 미구현 — 모바일에선 견본이 위, 7카드가 아래로 길게 쌓여 스크롤 지옥. 최소한 견본을 sticky 상단 축소바로.

### B2. "여지껏 없던 조합"의 독창성 미달 — 컨셉이 표면에 0 (마스터 최우선 요구 직격)
- **위치:** 전역. 특히 `landing.module.css`(히어로/band/howCard), `home.module.css`, 모든 카드 표면.
- **문제:** 디렉션 2-4가 차별점으로 내건 ① **한국 문방구(색지·스티커·마스킹테이프) 질감**, ② **"글자=살아있는 생물" 인터랙션**, ③ **붓획 모티프**가 시각적으로 전무하다. 현재는 "흰 카드 + 웜그레이 배경 + 코랄 악센트"라는 **표준 소프트 UI**일 뿐 — 색지(`--bg #f6f4f0`)는 거의 흰색과 구분 안 되고, 표면은 전부 순백 `#ffffff` 카드. "문방구"라 말만 하고 보여주지 않는다. `InteractiveSpecimen`도 `skewX`로 글자를 기계적으로 기울일 뿐(`InteractiveSpecimen.tsx:34-37`) "살아있는 생물" 느낌이 아니다.
- **개선 지시 (택1 이상 강하게 밀 것):**
  1. **색지(色紙) 물성 도입**: 카드/band를 순백 대신 **미세 종이 결 텍스처 + 따뜻한 색지 톤 2~3종**(버터·민트·자두 `--candy-*`를 *배경 면*에 옅게)으로. 지금 candy 변수는 마스코트 볼터치에만 쓰여 거의 죽어 있음.
  2. **"스티커/마스킹테이프" 디테일**: 프리셋 칩이나 견본 라벨을 **살짝 기울어진 스티커**(미세 `rotate(-2deg)` + 그림자)로 — 단번에 "문방구" 시그널이 박힌다. 절제해서 1~2곳만.
  3. **붓획 모티프**: 섹션 구분/CTA 밑줄을 직선 대신 **붓으로 그은 한 획(SVG stroke)** 으로. 로고 "획"과 마스코트 등의 붓이 따로 노는데, 이 붓획을 UI 장식 언어로 통일하면 "획"이라는 이름과 폰트앱 본질이 비로소 맞물린다.
  4. **"글자=생물" 인터랙션**: 슬라이더 조작 시 견본 글자가 `--ease-spring`으로 통통 반응(현재 디바운스 350ms 후 정적 교체뿐). 마스코트가 견본 변할 때 `surprised`로 반응하게 연결(현재 무드는 loading 여부로만).

## Warning (개선 권고)

### W1. 색지 `--bg`가 사실상 흰색 — clean이 과해 "밋밋"
- **위치:** `globals.css:18` `--bg:#f6f4f0`, 카드 `--surface:#ffffff`. 둘 차이가 명도상 미미해 "떠 있는 카드"감이 약하고 화면이 전체적으로 흰 종이 한 장. 색지 톤을 한 단 더 따뜻하게(예: 살짝 더 채도) 내려 카드와 대비를 줘야 depth가 산다.

### W2. 마스코트 너굴이 — 완성도 높으나 헤더에서 36px로 작게 소비, 캐릭터성 손해
- **위치:** `SiteChrome.tsx:16`(헤더 36px still), `page.tsx:45`(eyebrow 28px), `studio:325`(honesty 22px). SVG 자체는 우수(붓·꼬리·따봉·표정 6종 path 경량, 다크 대응). 다만 **헤더·eyebrow에서 너무 작아 "귀여움 체감"이 안 산다**(22~36px에선 표정 디테일 소실, round2 W4도 지적). 홈 meet 섹션 104px가 유일하게 캐릭터다움이 산다.
- **지시:** 헤더 마스코트를 44px+로 키우거나, 스튜디오 **빈/완성 순간에 큰 너굴이 + 말풍선**("오, 멋진데요!")을 등장시켜 감정 앵커로. 현재 완성(love)·에러(worried) 표정 path가 정의돼 있는데 **실사용처가 없다**(FontPreview는 sleepy/surprised만, `FontPreview.tsx:275`). 다운로드 성공 토스트에 love 너굴이를 띄워라 — 만들어 둔 자산을 안 씀.

### W3. 변주 갤러리 9분할 첫인상 약함 (오케스트레이터 시드)
- **위치:** `VariationGallery.tsx:157-200`, `.cell` inset 박스(`VariationGallery.module.css:17-31`).
- 버튼 눌러야 9칸이 pending→ready로 인라인 로딩. 첫 진입 시엔 트리거 버튼 하나만 보여 "이 앱이 뭘 해주는지" 한눈에 안 들어온다. 칸도 `surface-2` 회색 inset이라 9개 변형이 떠 있지 않고 가라앉아 보임.
- **지시:** 스튜디오 첫 로드 시 **자동 1회 생성**(또는 프리셋 클릭과 동시 갱신)으로 9칸을 채워 "와" 모먼트를. ready 칸은 inset이 아니라 살짝 떠 있는(shadow-sm) 카드로 — 고를 수 있는 후보임을 시각적으로.

### W4. Segmented 인디케이터 3+옵션에서 위치 오차 (round2 W3 미해결)
- **위치:** `Segmented.tsx:44-46` `transform:translateX(calc(${idx} * 100%))`. thumb 폭 `(100%-8px)/n`에 대해 100% 평행이동이라 트랙 padding(4px)+옵션 3개↑에서 누적 오차. 현재 2-옵션(라틴/한글, WOFF/TTF)뿐이라 회귀 아님 → 향후 문자체계 추가 시 깨짐. `translateX(calc(${idx} * (100% + 8px/${n})))` 식 보정 권장.

### W5. 모션 어휘 빈약 — 스프링은 토큰뿐, 실 적용은 누름 scale 위주
- `--ease-spring` 정의돼 있으나 실제로는 버튼/칩/thumb의 `:active scale`과 마스코트 float뿐. 디렉션이 약속한 "시트 슬라이드업", "글자 스프링 반응", "진입 스태거"가 없어 iOS의 "살아있는" 물성 체감이 약하다. 견본 등장/프리셋 적용 시 `--ease-spring` 진입 모션 1~2개만 추가해도 인상이 달라진다(reduced-motion은 이미 잘 처리됨).

### W6. InteractiveSpecimen / 홈 SAMPLES가 시스템 글꼴 skew 흉내 — 첫인상 진정성 (round2 W6)
- **위치:** `InteractiveSpecimen.tsx:34-37`, `home/app/page.tsx:62-66`. 폰트앱 히어로의 주인공이 실제 생성 폰트가 아니라 시스템 글꼴을 `skewX`로 기울인 것. aria-label에 정직히 "시스템 글꼴 흉내" 명기한 건 좋으나, **히어로에서 진짜 엔진 결과를 1회라도 보여주면** 신뢰·독창성 둘 다 크게 오른다.

### W7. `--ink-faint`가 다수 캡션에 광범위 사용 — 경계선 대비
- **위치:** groupHead(`FontStudio.module.css:54-58`), subhead, seedLabel, band/sample/next 라벨, footer fineprint 등. `#6b6875` on `--surface-2(#f0ece6)`는 round2가 4.7:1로 통과 보고했으나 caption(0.78rem) 다수에 쓰여 **저시력 사용자에겐 흐림**. 정보성(슬라이더 값은 칩으로 OK) 라벨은 `--ink-soft`로 한 단 올릴 것을 권장.

### W8. 디스플레이 폰트 Quicksand가 한글 미지원 — 한글 제목은 폴백
- **위치:** `layout.tsx:16-21` Quicksand `subsets:["latin"]`. `.display` 클래스가 붙은 한글 제목("글씨체를 빚는 작업대" 등)은 Quicksand가 한글 글리프가 없어 Noto Sans KR로 폴백 → **의도한 "둥근 귀여운 디스플레이"가 한글에선 안 먹힌다.** 한글 라운드 디스플레이 대안(예: 둥근 한글 웹폰트) 검토하거나, display는 라틴 로고에만 한정.

## 항목별 점검 요약
- **iOS 트렌드 충실도:** 세그먼티드·소프트섀도·클레이 thumb·블러 머티리얼 헤더·연속곡률·44px 터치·focus-visible·다크모드·reduced-motion 전부 실재. **기본기는 A급.** 단 바텀시트/탭바(모바일) 미구현으로 모바일 iOS 정합은 미완.
- **cute/clean 균형:** clean 우세 확실. cute는 마스코트로 분리 주입 성공이나 **표면(색지·스티커·붓획)에 cute가 안 배어 "차가운 클린"에 가깝다** → B2 참조.
- **독창성("여지껏 없던 조합"):** 현재 **미달**. 컴포넌트 조합이 표준 소프트 UI라 "어디서 본 듯". 컨셉(문방구·획·생물)을 표면화해야 마스터 요구 충족.
- **브랜드 일관성:** 워드마크 "획" + 마스코트 너굴이 + 정직성 라벨은 일관. 다만 "획(붓획)"이라는 이름값이 UI 어휘로 안 쓰여 이름과 화면이 따로 논다.
- **다크/대비/모션:** 다크 토큰·대비(AA 통과)·reduced-motion은 견고. 모션 적용 범위만 빈약(W5).
