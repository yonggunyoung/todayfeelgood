# 10. 포인트·미니게임·토스 입점 — 설계와 운영

> 목표: 절약 "행동"을 보상해 매일 열어보게 만들고(리텐션), 짬시간에 광고 수익을 만든다.
> 같은 코드가 **개별 운영(웹/PWA)** 과 **앱인토스(토스 미니앱)** 양쪽에서 돈다.

## 1. 포인트 경제 (js/points.js)

원칙: **무상 적립만** (현금 충전 없음 → 선불전자지급수단 규제 비대상), 현금 환급 없음,
일일 상한으로 어뷰징 차단, 교환은 내부 보상(원가≈0)부터.

| 적립처 | P | 일일 상한 | 비고 |
|---|---|---|---|
| 출석 | 5 | 1회 | 앱 첫 실행 시 자동 |
| 요리 완료 | 10 | 3회 | 재고 차감 확정 시 |
| 임박 재료 구출 | 20 | 3회 | D-3 이하 재료를 요리에 사용 |
| 잔반 해결 | 15 | 3회 | 잔반 "먹었어요" |
| 광고 시청 보너스 | 3 | 3회 | AI 충전 광고 완주에 얹어줌 |
| 미니게임 | 점수 기반 1~12 | 5판 | 판수 초과 시 기록 도전만 가능 |

→ 적립은 전부 **무상**(현금 유입 없음)이라 그 자체로 비용이 들지 않는다. 비용은 "교환할 때"만 발생.

### 교환소 (SHOP) — 사용처를 수익성에 맞춰 확정

| 상품 | P | 실제 원가 | 가드 | 구현 |
|---|---|---|---|---|
| AI 1회권 | 100 | 스캔 ≈ ₩7 / 영상정리 ≈ ₩40~100 | 서버 `/redeem` **일 2회**(REDEEM_DAILY_CAP) | 무료 한도 +1 |
| 광고 없는 하루 | 300 | ₩0 (자체 배너만 미노출) | 24시간 | `adFreeUntil` |
| 프리미엄 맛보기 1일 | 1,000 | ₩0 | 24시간·구독 전환 미끼 | `planTrialUntil` |
| 기프티콘 (잠금) | 5,000 | 실비(원가) | P1·본인확인 | B2B 발행사 |
| 토스포인트 전환 (잠금) | — | 실비 | P2 | 앱인토스 프로모션 |

### 수익 대비 계산 (2026 국내 기준 보수적 추정)

| 항목 | 추정치 | 메모 |
|---|---|---|
| 보상형 광고 1회 완주 매출 | **₩10~15** | AdMob/토스 리워드 eCPM ₩10,000~15,000/1,000완주 기준 |
| 배너 노출 1회 매출 | ₩0.5~2 | AdFit/애드센스, 슬롯 3곳 |
| 보상형 1회로 주는 것 | AI +1회(원가 ₩7) **또는** 게임 2배 P(₩0) | 둘 다 매출 > 원가 → **흑자** |
| 포인트→AI 1회권 (광고 없이) | 원가 ₩7, 매출 ₩0 | **유일한 순지출** — 서버 일 2회로 상한 = 최대 ₩14/일/인 |

**핵심**: 광고를 거치는 보상은 전부 흑자(매출 ₩10~15 > 원가 ≤₩7). 광고 없이 포인트로만 바꾸는 AI 1회권만
순지출인데, 서버가 하루 2회로 막아 1인당 최대 ₩14/일 — 게다가 그 유저는 매일 출석·게임으로 광고를 봐
이미 그 이상을 벌어준 상태다. **현금 환급은 없음** → 포인트가 외부로 빠져나가지 않는다.

예산 수칙(외부 교환 P1 도입 시): **기프티콘 예산 = 전월 광고 순매출의 15~20% 상한**. 토스 비즈월렛처럼
예산 소진 시 교환 자동 마감. 이 비율이면 무조건 흑자 구조가 유지된다.

## 2. 미니게임 5종 (짧고, 중독성, 승부욕)

| 게임 | 파일 | 방식 | 중독 장치 |
|---|---|---|---|
| 🧊 냉장고 지키기 | game-defense.js | 캔버스 웨이브 디펜스 | 콤보·보스(곰팡이대왕)·파워업(❄️💣❤️)·파티클·화면흔들림·위험 비네트 |
| 🍎 재료 매치 | game-puzzle.js | 캔버스 매치3 | 3매치+연쇄 콤보, 60초 스피드 |
| 🥬 프레시 캐치 | games.js | 원터치 타이밍 | 콤보 ×5, 라운드마다 가속·존 축소, 생명 3 |
| 🧠 냉장고 상식 퀴즈 | game-quiz.js | 음성/탭 | 신기한 음식 상식 14문, 틀리면 광고로 정답 or 스킵 |
| 🎤 외쳐! 재료 | games.js | 음성 스피드 | 8문제, 초성 힌트, 패스 페널티 (iOS 잠금) |

- 디펜스/퍼즐은 `<canvas>` + DPR 스케일 + WebAudio 합성 효과음·햅틱 — **단독 배포 가능한 완성도** 지향
- 캐릭터: 팔다리 없는 "눈 중심" 아트(브랜드 일관) — 말랑 슬라임 바디 + 깜빡이는 눈 + 상징 이모지 배지

수익 연결 (전부 `playAd` 코어 경유 — 광고망 교체는 코드 1곳):
- 판 종료 → 포인트 적립 → **[📺 광고 끝까지 보고 2배]** (완주해야 지급)
- 입구가 곳곳에: 상단바 🅿 배지, 홈 포인트 카드, **요리 타이머 위젯**, 긴 조리 단계의 "⏳ N분 — 타이머·게임"
- 점수→포인트 환산(PTS_DIV)으로 게임별 보상 균형, **일 5판** 보상 제한(무한 파밍 차단)

### 랭킹 (sync.js submitScore/topScores)
- `leaderboard/{uid}` = `{ name, photo, code(가족), best:{게임별 최고}, total }`
- 🌍 전체(게임별 orderBy) + 👨‍👩‍👧 우리 냉장고(같은 가족 코드 where) — 로그인 사용자만 참여
- **Firestore 보안 규칙** 추가 필요(docs/07): `leaderboard`는 로그인 사용자 읽기 허용 + 본인 문서만 쓰기

```
match /leaderboard/{uid} {
  allow read: if request.auth != null;
  allow write: if request.auth != null && request.auth.uid == uid;
}
```

## 3. 토스 입점 어댑터 (js/toss.js)

같은 빌드가 토스 안에서 자동 전환된다 — SDK 전역이 감지되면:

| 기능 | 개별 운영 (지금) | 앱인토스 |
|---|---|---|
| 보상형 광고 | 하우스 15초 (AdFit 교체 자리) | `showFullScreenAd` → `userEarnedReward` 수신 시 보상 |
| 포인트 | 앱 내 포인트 | 앱 내 포인트 + **토스포인트 프로모션 미러링** (`tossGivePoints`) |
| 결제 | 토스페이먼츠 (G2) | 토스 인앱결제 SDK |

### 입점 체크리스트 (운영자)

1. 앱인토스 콘솔 가입 (만 19세+, 본인 명의 토스앱 — 사업자 없이 출시 가능)
2. 미니앱 등록: 이름 냉비서 / 카테고리 라이프 / 개인정보처리방침 URL(blog/privacy.html) / 고객센터 채널
3. 콘솔에서 **보상형 광고 단위** 생성 → `js/config.js` `TOSS.rewardAdId`
4. 콘솔에서 **프로모션(토스포인트)** 생성 + **비즈월렛 예산 선충전** → `TOSS.promotionId`
   - 지급 실비만 차감, 플랫폼 수수료 없음 · 예산 80% 소진 시 메일 알림 · 소진 시 지급 자동 중단
5. WebView SDK에 앱 URL 연결 → 검수 제출 (비게임 1~3개월)
6. SDK 전역 객체/함수명은 입점 시점 문서로 확정 → `js/toss.js`의 후보 명칭 수정 (30분 작업)

## 4. 법·운영 체크 (요약)

- 적립형 포인트(무상)·내부 보상 교환: 규제 부담 없음. **현금 충전/환급은 도입하지 않는다.**
- 기프티콘 등 경품: 1인당 5만원 초과 시 제세공과금(기타소득 22%) 이슈 — 소액 상품만.
- 어뷰징: 일일 상한(클라) + `/redeem`·`/reward` 서버 상한(일 2/3회) 이중. 외부 교환 도입 시 본인인증 추가.
- 게임은 실력 기반(사행성 아님). 룰렛 등 추첨형을 추가할 땐 "꽝 없음 소액"으로 설계.
