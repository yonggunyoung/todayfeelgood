# 자동 배포 설정 (GitHub Actions → 오라클)

GitHub Actions의 **Deploy** 워크플로우가 서버에 SSH로 접속해
`git fetch/checkout → docker compose up --build` 까지 자동으로 해줍니다.
한 번 설정해 두면 **Actions 탭에서 버튼 한 번**으로 배포돼요.

> 자동(push 시 자동 배포) 아님 — 무료티어 보호를 위해 **수동 실행(버튼)** 방식입니다.

---

## 0. 전제 (서버 1회 준비)

오라클 서버에 레포가 클론돼 있고 docker가 깔려 있어야 합니다. 처음이면 서버에서:

```bash
# 예: /home/ubuntu/webapp 에 클론
cd ~
git clone <이 레포의 git URL> webapp
cd webapp
cp .env.example .env   # 필요한 값 채우기(.env 는 커밋 금지)
docker compose up -d --build   # 최초 1회 수동으로 떠는지 확인
```

- `docker` / `docker compose` 설치돼 있어야 함.
- 보안목록/방화벽에서 **80, 443 포트** 열려 있어야 함.

## 1. Actions가 쓸 SSH 키 준비

```bash
ssh-keygen -t ed25519 -C "gha-deploy" -f gha_deploy   # 키페어 생성(이미 있으면 재사용)
```

- 공개키(`gha_deploy.pub`)를 **서버**의 `~/.ssh/authorized_keys` 에 추가:
  ```bash
  cat gha_deploy.pub >> ~/.ssh/authorized_keys   # 서버에서 실행
  ```
- 개인키(`gha_deploy` 파일 전체 내용)는 아래 시크릿으로 등록.

## 2. GitHub 시크릿 4개 등록

레포 → **Settings → Secrets and variables → Actions → New repository secret**:

| 이름 | 값 |
|---|---|
| `ORACLE_HOST` | 서버 공인 IP 또는 도메인 |
| `ORACLE_USER` | SSH 사용자 (예: `ubuntu`) |
| `ORACLE_SSH_KEY` | 개인키 PEM **전체 내용** (`gha_deploy` 파일 통째로) |
| `DEPLOY_PATH` | 서버의 레포 경로 (예: `/home/ubuntu/webapp`) |

## 3. 배포 실행

레포 → **Actions 탭 → "Deploy" → Run workflow** → `ref` 입력:

- 지금 작업 브랜치로 바로 올리기: `claude/great-darwin-1FmUy`
- (나중에) main 병합 후: `main`

→ **Run workflow**. 서버가 해당 ref로 맞춘 뒤 docker를 재빌드합니다.

## 4. 확인

- Actions 로그 맨 끝 `docker compose ps` 로 컨테이너 상태 확인.
- `https://ddukkit.com` 접속 → 새 화면(뚝딱 허브) 확인.

---

## ⚠️ 오라클 무료티어 주의 (메모리)

`docker compose up --build` 가 한 번에 여러 서비스(home·font·sticker·sign·kit·textmoji·nginx)를
빌드하면 **메모리 부족(OOM)** 으로 죽을 수 있어요. OOM이 나면 서버에서 **나눠 빌드**:

```bash
cd <DEPLOY_PATH>
docker compose build home && docker compose up -d home
docker compose build font && docker compose up -d font
# ... 서비스별로 반복
```

또는 스왑(swap) 메모리를 잡아두면 한 번에 빌드가 수월해집니다.
